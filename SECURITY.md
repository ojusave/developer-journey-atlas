# Security policy

## Supported version

Security fixes apply to the current `main` branch. No released version is currently supported as a deployed production service.

## Reporting a vulnerability

Use GitHub's private vulnerability reporting for this repository. Open the repository's Security tab, select **Advisories**, then select **Report a vulnerability**.

Do not include vulnerability details, credentials, participant data, customer data, or private company information in a public issue.

Include the affected path or component, reproduction steps, impact, and any suggested mitigation. No response or remediation time is guaranteed.

## Research-data concerns

For exposed personal information, unsupported causal claims, incorrect attribution, or third-party rights concerns in the research corpus, use the same private reporting path and identify the exact record and field.
