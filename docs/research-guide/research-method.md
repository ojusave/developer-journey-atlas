# Research method

## Bounded question

For a named platform and one documented developer goal, reconstruct the official path from the documented entry point to an explicitly named or demonstrated first success.

This is documentation research. It is not telemetry, usability testing, conversion analysis, or proof of abandonment.

## Evidence order

1. Current official documentation and API references.
2. Current official quickstarts, account guides, support pages, and example repositories.
3. Direct observation of publicly accessible states.

Search results and model memory can locate sources but cannot support a recorded fact. Secondary sources may identify questions but do not establish the canonical route.

## Recording rules

- Preserve official order, prerequisites, choices, branches, waits, and success signals.
- Attach source IDs to evidence-bearing fields.
- Record undocumented transitions as uncertainties.
- Record only vendor-stated time.
- Stop at the selected first-success boundary.
- Do not turn a documented requirement into a confirmed drop-off.
- Do not turn a blocker hypothesis into a cause without evidence from a real attempt.

The imported corpus retains its original detailed method in `packages/journey-corpus/agent-instructions.md`, `SELECTION-POLICY.md`, and `MEASUREMENT-CONTRACT.md`.

## Research freshness

`researched_at` and each source's `accessed_at` describe the snapshot. A record is not current merely because it validates. Recheck official sources before using a record for a consequential current claim.
