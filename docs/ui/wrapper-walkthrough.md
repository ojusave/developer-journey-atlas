# Wrapper walkthrough

## One-screen promise

Search a developer platform and see the documented path to first success, where its route differs from qualified peers, and what deserves investigation.

## Information order

1. Search.
2. First-success outcome.
3. Required actions, waits, decisions, and gates.
4. Anonymous peer context.
5. Investigation prompts.
6. Full route and sources on demand.

This order keeps the first screen useful without forcing a reader through every research detail.

## Comparison state

The comparison is available only with at least three qualified peers. The wrapper shows each component value and cohort median, plus a count of components above the median. It does not expose peer names, a weighted effort score, or a leaderboard.

## Missing-platform state

The visitor can start live research without leaving the result page. Progress streams in short steps. The draft is visibly unverified, its official sources remain attached, and any GitHub contribution is a draft pull request.

## Accessibility baseline

- semantic headings and landmarks;
- labeled search combobox;
- arrow-key autocomplete navigation;
- visible focus styles;
- live result announcements;
- mobile layout down to 320 pixels;
- reduced-motion support;
- disclosure controls for long routes and source lists.

These implementation checks do not replace testing with people who have ADHD or other access needs.
