# Contributing to Developer Journey Atlas

Developer Journey Atlas is evidence-grounded. Contributions should make the research easier to verify, not make its claims sound stronger than the evidence allows.

## Research contributions

- Preserve source URLs, access dates, evidence status, and provenance fields.
- Do not present blocker hypotheses as observed causes of drop-off.
- Do not infer prevalence, ranking, or causal impact from the current corpus.
- Keep platform records human-readable and valid against the repository schemas.
- Submit research changes for review. Generated output is not a canonical source.

## Software contributions

- Keep changes focused and explain the user or research outcome.
- Add or update tests for behavior changes.
- Do not commit secrets, customer data, or private incident details.
- Run `npm run test:all`, `npm run build`, and `npm --prefix workflows run build` before requesting review.
- Regenerate derived views when canonical records or taxonomy content changes.

## Licensing contributions

By submitting a contribution, you agree that software contributions are provided under Apache-2.0 and original research contributions are provided under CC BY 4.0, according to `LICENSE_SCOPE.md`. Only submit material you have the right to contribute. Third-party material must retain its source and applicable rights information.
