**Developer goal:** Deploy a first PostgreSQL database on PlanetScale and obtain connection credentials.

**Meaningful success:** A PlanetScale Postgres database with a main branch is provisioned, and the developer has generated and recorded a default role plus a usable connection string.

**Evidence boundary:** explicitly-named. It is the terminal outcome explicitly stated by the official quickstart before the separate Day one and production-readiness material.

## Documented path

1. Create or log in to the PlanetScale account, complete email confirmation when registering, and select the target organization. Success signal: The selected organization is visible. Sources: S1, S4.
2. Open Organization Settings, Billing and add a valid debit or credit card before choosing a database plan. Success signal: Billing shows the payment method and database creation can proceed. Sources: S5.
3. Click New database and select the Postgres engine. Success signal: Postgres is the selected engine. Sources: S1.
4. Enter a unique database name, choose a primary region, then select cluster size and CPU architecture. Success signal: All required configuration fields are populated. Sources: S1.
5. Review the settings and click Create database. Success signal: The new database and default main branch appear in the dashboard. Sources: S1.
6. Open the new database, click Connect, select Default role, and click Create default role. Success signal: The Connect flow displays Host, Username, and Password. Sources: S1.
7. Record the host, username, password, database, and port, then generate a connection string under How are you connecting?. Success signal: The credentials and connection string are visible and securely recorded; the guide then states that the database is ready. Sources: S1.

## Documented friction gates

- Account and organization creation precede database creation. Stage 1. Evidence state: documented requirement. Sources: S1, S4.
- All plans require an organization payment method; prepaid cards are rejected and card verification can create a temporary hold. Stage 2. Evidence state: documented requirement. Sources: S5.
- The database creation form requires a choice between Vitess and Postgres. Stage 3. Evidence state: documented requirement. Sources: S1.
- Cluster size and CPU architecture are required configuration choices; PlanetScale's Postgres product page says single-node Postgres starts at $5 per month. Stage 4. Evidence state: documented requirement. Sources: S1, S2.
- A privileged default role must be generated, and its one-time-displayed password must be stored securely. Stage 6. Evidence state: documented requirement. Sources: S1.
- The connection format, direct versus PgBouncer port, and required SSL parameters must be selected for the intended client. Stage 7. Evidence state: documented requirement. Sources: S1.
