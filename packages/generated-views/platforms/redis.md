## Documented path

1. Open the Redis getting-started documentation and follow Redis Cloud quick start. Success signal: The page lists Create an account and a free database, then Connect to your database. Sources: S1, S2.
2. Open the Redis Cloud Sign up page. Success signal: The form and social signup options are visible. Sources: S2.
3. Create the account with the form or sign up with Google or GitHub. Success signal: Redis sends an activation email. Sources: S2.
4. Open the Redis activation email and select Activate account. Success signal: The Redis Cloud console opens the database-creation page. Sources: S2.
5. Keep the Free plan selected on the Create database page. Success signal: Free is selected. Sources: S2.
6. Accept the generated database name or change the Database name field. Success signal: A database name is present. Sources: S2.
7. Select the Database version. Success signal: A database version is selected. Sources: S2.
8. Choose a Cloud vendor and Region. Success signal: Both fields are selected. Sources: S2.
9. Select Create database. Success signal: The Database details page appears. Sources: S2.
10. Wait for the database status to change from pending to active. Success signal: The status indicator becomes a green circle with a checkmark. Sources: S2.
11. In Configuration details, go to Security and reveal the Default user password. Success signal: The default username and password are available for connection. Sources: S2.
12. Select Connect to open the connection wizard. Success signal: The connection methods are displayed. Sources: S2.
13. Under Redis Insight, select Launch Redis Insight web. Success signal: Redis Insight opens in a new tab for the database. Sources: S2.

## Documented friction gates

- A new account requires form or social signup followed by activation through email. Stage 3. Evidence state: documented requirement. Sources: S2.
- The developer must choose a database version, cloud vendor, and supported region; no defaults are prescribed by the quick start. Stage 8. Evidence state: documented requirement. Sources: S2.
- Database creation has a processing pause and pending state before the active checkmark appears. Stage 10. Evidence state: documented requirement. Sources: S2.
- Database connection credentials are protected behind Default user, Configure, and password-reveal controls. Stage 11. Evidence state: documented requirement. Sources: S2.
- The connection wizard offers GUI, CLI, and multiple language-client routes with different installation and execution requirements. Stage 13. Evidence state: documented requirement. Sources: S2, S3.
