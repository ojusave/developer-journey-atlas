**Meaningful success:** The developer's first server-side web app has a Live Render deploy and serves its root content from the assigned onrender.com URL.

## Documented friction gates

- The tutorial requires a deployment-interface choice among Dashboard, CLI, and coding agent without declaring one preferred route. Stage 2. Evidence state: documented requirement. Sources: S2.
- A Render account is required before the developer can use Account Settings and create a service. Stage 3. Evidence state: documented requirement. Sources: S2.
- The developer must authorize Render's GitHub access and include the app repository. Stage 6. Evidence state: documented requirement. Sources: S5.
- The developer must classify the app as Web Service or Static Site based on whether it contains server-side code. Stage 7. Evidence state: documented requirement. Sources: S2.
- The linked deployment credential must have access to the repository selected for deployment. Stage 10. Evidence state: documented requirement. Sources: S4, S5.
- The service creation flow requires a region choice, and the first-deploy docs do not prescribe one. Stage 12. Evidence state: documented requirement. Sources: S4.
- The selected runtime must match the application or its Dockerfile. Stage 14. Evidence state: documented requirement. Sources: S2, S4.
- The developer must provide an application-specific build command. Stage 15. Evidence state: documented requirement. Sources: S2, S4.
- The developer must provide an application-specific start command. Stage 16. Evidence state: documented requirement. Sources: S2, S4.
- The developer must choose an instance type; Free avoids payment for this tutorial but has documented limitations. Stage 17. Evidence state: documented requirement. Sources: S2, S4.
- The developer waits for Render to build, start, and evaluate the first deploy; no numeric duration is documented. Stage 20. Evidence state: documented requirement. Sources: S2.
- A Web Service must bind to a public HTTP port on 0.0.0.0 or the deploy fails. Stage 22. Evidence state: documented requirement. Sources: S4.
