3. Enter address and phone details, then add an accepted credit or eligible debit card and select Finish. Success signal: The payment verification finishes and the trial terms are available. Sources: S4, S5.
4. Accept the trial terms and select Start my free trial. Success signal: Oracle begins provisioning cloud services in the tenancy. Sources: S4, S5.
5. Wait for tenancy provisioning and the redirect to the OCI Console, then retain the welcome email. Success signal: The OCI Console opens and the welcome email confirms the account. Sources: S4.
6. Open the OCI documentation home and follow its Start route to first-resource tutorials. Success signal: The developer reaches the documented first-resource choices. Sources: S1, S2.
7. Select compute, storage, networking, database, load balancing, or another resource path and its interface. Success signal: No common platform-level terminal signal is documented. Sources: S2, S3.
8. Open Oracle Cloud Free Trial signup, enter country or territory, name, valid email, and CAPTCHA, then select Verify my email. Success signal: The verification email arrives with a signup link. Sources: S4.
9. Open the verification link, create and confirm the password, optionally enter a company name, and choose a cloud account name. Success signal: The signup form accepts the account information. Sources: S4.
10. Select the Home Region, accept the terms, and continue. Success signal: The address and phone section opens. Sources: S4, S5.
11. Enter the address and phone details, provide a post office box if Oracle asks for one, provide a CPF if the address is in Brazil, and select Continue. Success signal: The Add payment verification method control becomes available. Sources: S4.
12. Select Add payment verification method, choose Credit Card, enter an accepted card's details, and select Finish. Success signal: Payment verification completes and the trial terms become available. Sources: S4, S5.
13. Accept the trial terms and select Start my free trial. Success signal: Oracle begins provisioning cloud services. Sources: S4, S5.
14. Wait for tenancy provisioning and the redirect to the OCI Console, then retain the welcome email. Success signal: The OCI Console opens and the welcome email confirms the account. Sources: S4.
15. Confirm that the local macOS, Linux, or Windows computer has an `ssh` client. Success signal: The `ssh` command is available locally. Sources: S6.
16. Open Identity & Security, select Compartments, create a compartment with a name, required description, and parent compartment, then select it. Success signal: The new compartment appears in the list and can be selected. Sources: S6.
17. Open Networking, select Virtual cloud networks, choose Start VCN Wizard, and select Create VCN with Internet Connectivity. Success signal: The VCN configuration form opens for the selected compartment. Sources: S6.
18. Name the VCN, keep the tutorial defaults for DNS and public and private subnet address settings, review the selections, and select Create. Success signal: OCI begins creating the VCN and its related resources. Sources: S6.
19. Wait for VCN creation to complete and select View VCN. Success signal: The VCN details page shows the created network resources. Sources: S6.
20. Open Compute, select Instances and Create Instance, then enter the instance name and select the tutorial compartment. Success signal: The instance configuration form accepts the name and compartment. Sources: S6.
21. Keep the tutorial placement and security defaults, select the Oracle Linux image and an Always Free eligible virtual-machine shape, then select the created VCN and its public subnet with a public IPv4 address. Success signal: The form shows the selected image, shape, VCN, public subnet, and public IPv4 assignment. Sources: S6.
22. Use Generate a key pair for me, save both private and public SSH keys in a secure location, keep the documented boot-volume and no-block-volume settings, and select Create. Success signal: The instance details page opens in Provisioning state. Sources: S6.
23. Wait without refreshing until the instance reaches Running, then allow several additional minutes for Oracle Linux to boot and obtain its public IP address. Success signal: The lifecycle state is Running and the public IP is visible. Sources: S6.
24. Restrict the private key permissions, then run `ssh -i <private_key_file> opc@<public-ip-address>` and accept the host-key fingerprint on first connection. Success signal: SSH accepts the key and opens the remote session. Sources: S6.
25. Verify that the default Oracle Linux shell prompt is visible on the remote instance. Success signal: The developer is connected to the default shell for the instance. Sources: S6.

## Documented friction gates

- Email verification is mandatory and its signup link expires after 30 minutes. Stage 1. Evidence state: documented requirement. Sources: S4.
- Home Region must be selected during signup and cannot be changed later. Stage 3. Evidence state: documented requirement. Sources: S4, S5.
- Most users must provide a payment card, and a temporary verification hold may appear. Stage 4. Evidence state: documented requirement. Sources: S4, S5.
- Tenancy services must be provisioned before the OCI Console redirect and welcome email. Stage 6. Evidence state: documented requirement. Sources: S4.
- OCI requires a resource and interface choice among several equally presented first-resource paths. Stage 8. Evidence state: documented requirement. Sources: S1, S2.
