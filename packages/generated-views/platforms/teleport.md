## Documented path

1. Open the Community Edition deployment guide and select Local Docker container. Success signal: The Docker prerequisites load. Sources: S4.
2. Install mkcert, create `teleport-tls`, install the local CA, generate a localhost certificate, and copy `rootCA.pem` into the directory. Success signal: `localhost.pem`, `localhost-key.pem`, and `rootCA.pem` are present. Sources: S4.
3. Start the documented Ubuntu container with the TLS directory mounted and port 3080 mapped to 443. Success signal: The container shell opens. Sources: S4.
4. Install curl, trust the copied mkcert CA, and run the official Teleport install script inside the container. Success signal: The `teleport` command is available. Sources: S4.
5. Run `teleport configure` with cluster name localhost, public address localhost:443, and the mounted certificate and key. Success signal: Configuration completes without error. Sources: S4.
6. Run `teleport start --config="/etc/teleport.yaml"`. Success signal: The process remains running and listens through mapped port 3080. Sources: S4.
7. Open `https://localhost:3080`, accept the Community Edition terms, and verify the welcome screen. Success signal: The guide's Verify Teleport is running checkpoint passes. Sources: S4.
