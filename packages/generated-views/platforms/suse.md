**Meaningful success:** The K3s install script starts a single-node Kubernetes cluster and installs a configured `kubectl`.

**Evidence boundary:** demonstrated-terminal-state. A Ready node is the first observable proof that the K3s service, control plane, datastore, kubelet, and container runtime are running.

## Documented path

1. Open the official K3s Quick Start. Success signal: The Install Script section loads. Sources: S5.
2. On a requirements-compatible Linux host, run `curl -sfL https://get.k3s.io | sh -`. Success signal: The install command exits successfully. Sources: S5.
3. Run `sudo k3s kubectl get nodes` and verify the local node is Ready. Success signal: The single node reports Ready. Sources: S5.
