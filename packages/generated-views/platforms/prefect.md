## Documented path

1. Open Prefect Getting Started and select Quickstart. Success signal: The page frames the tutorial around code, execution location, and timing. Sources: S1, S3.
2. Choose Cloud or Open Source before setting up the API backend and running the shared example. Success signal: The chosen tab's setup instructions are visible. Sources: S1.
3. Create and activate a Python 3.10-or-newer virtual environment if needed. Success signal: The intended environment is active. Sources: S2.
4. Run `pip install -U prefect`. Success signal: pip completes the installation. Sources: S2.
5. Run `prefect version`. Success signal: The command prints the installed Prefect version information. Sources: S2.
6. Run `prefect server start` and keep the process running. Success signal: The server is available at `http://localhost:4200`. Sources: S1.
7. In a new terminal, run `git clone https://github.com/PrefectHQ/quickstart && cd quickstart`. Success signal: The shell is in the cloned `quickstart` directory with `01_getting_started.py`. Sources: S1.
8. Run `python 01_getting_started.py`. Success signal: The terminal prints a Prefect UI graph link followed by state changes and logs, and the Quickstart then states that the Prefect flow has run successfully locally. Sources: S1.
