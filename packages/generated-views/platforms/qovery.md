## Documented friction gates

- The public docs establish identity providers and Terms acceptance but do not expose the atomic signup and verification sequence. Stage 2. Evidence state: documented requirement. Sources: S4, S5, S7.
- The signup guide says a first project is created during signup, while the first-app guide tells the same user to create a project after prerequisites. Stage 3. Evidence state: documented requirement. Sources: S3, S4.
- The free route needs substantial local Docker capacity and several installed utilities. Stage 4. Evidence state: documented requirement. Sources: S2.
- Qovery warns that its convenience CLI installation script does not verify the binary's integrity; a pinned manual download with checksum verification is the documented safer alternative. Stage 5. Evidence state: documented requirement. Sources: S2.
- Local k3s and Qovery component installation takes about 5 to 10 minutes and can prompt for administrator access on macOS and Windows. Stage 8. Evidence state: documented requirement. Sources: S2.
- Repository access can require broad provider permissions or a separately created organization token before the repository is selectable. Stage 13. Evidence state: documented requirement. Sources: S3, S5.
- The general first-app guide does not provide a canonical starter repository, so code, Dockerfile, dependencies, branch, and root path depend on the developer's own artifact. Stage 14. Evidence state: documented requirement. Sources: S3.
- The user must know the actual application port and all-interface binding; a mismatch prevents successful routing or health checks. Stage 16. Evidence state: documented requirement. Sources: S3, S6.
- Use build, deployment, and application logs to distinguish Dockerfile failures, resource exhaustion, crashes, and health-check failures, correct the cause, and redeploy. Stage 19. Evidence state: documented requirement. Sources: S3, S6.
- A Running deployment can still require 2 to 3 minutes of DNS propagation before its generated URL is reachable. Stage 20. Evidence state: documented requirement. Sources: S3.
