## Documented friction gates

- An Upsun account is required, but the linked public signup page's current atomic sequence is not documented. Stage 6. Evidence state: documented requirement. Sources: S1, S13.
- A first organization must be created and this starts the time-limited trial. Stage 7. Evidence state: documented requirement. Sources: S2, S7.
- The quickstart uses CLI creation while the full guide labels Console creation recommended. Stage 8. Evidence state: documented requirement. Sources: S1, S3.
- Framework or runtime and optional services can require interactive choices during initialization. Stage 10. Evidence state: documented requirement. Sources: S4.
- Generated YAML is only a starting configuration and can require app-specific correction before deployment. Stage 11. Evidence state: documented requirement. Sources: S4.
- Resources allocated to the app and services determine usage charges after the no-card trial and are limited during trial. Stage 12. Evidence state: documented requirement. Sources: S7, S12.
- A missing `.upsun` directory or invalid configuration aborts the first build. Add or correct the required files and push again. Stage 14. Evidence state: documented requirement. Sources: S4.
- Upsun validates, builds, allocates, deploys, starts, and routes traffic before the live URL is ready. Stage 15. Evidence state: documented requirement. Sources: S4, S5.
- An Active environment can still return an application error. Use official 502 and development troubleshooting if the primary URL does not serve the intended app. Stage 16. Evidence state: documented requirement. Sources: S6.
