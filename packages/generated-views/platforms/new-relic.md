**Developer goal:** For the Node.js candidate, instrument an existing server application and see its first traffic data in New Relic APM.

**Meaningful success:** The New Relic Node.js agent loads before the existing application, reports generated request traffic under its configured app name, and the data appears in New Relic APM.

**Evidence boundary:** demonstrated-terminal-state. Committed under workshop fastest/most-used policy; terminal taken from the reconstructed official path.

## Documented path

1. Open the New Relic signup page, enter Company Email and Name, accept the Terms of Service and Services Privacy Notice by submitting Get Started Free, or use Google, GitHub, GitLab, or Bitbucket signup. Success signal: New Relic sends the new-account confirmation email and creates an initial account with a license key. Sources: S1, S3, S5, S6, S10.
2. Open the New Relic confirmation email and complete account setup, then sign in. Success signal: The New Relic UI and account data are accessible. Sources: S6.
3. Choose the representative Node.js APM manual-install route instead of the general dynamic guided installer. Success signal: The eight-step manual installation procedure is selected. Sources: S1, S2, S4.
4. Check the compatibility table for the application's operating system, SHA-2 capability, Node.js and agent-version pairing, and every framework or module expected to be auto-instrumented. Success signal: The operating system and security requirements pass, the selected agent supports the Node.js release, and each relied-upon module version is inside its listed support range or has a planned manual instrumentation route. Sources: S1, S11.
5. In the application directory, run npm install newrelic. Success signal: The newrelic module is installed and present under node_modules. Sources: S1, S9.
6. Copy node_modules/newrelic/newrelic.js into the application root. Success signal: newrelic.js is beside the application root files. Sources: S1, S7.
7. Open the API keys UI and obtain a full Ingest - License key the developer is permitted to use, creating one if needed and storing it securely when shown. Success signal: A full 40-character license key is securely available for configuration. Sources: S5.
8. In newrelic.js, set license_key to the New Relic license key and app_name to a meaningful unique application name. Success signal: newrelic.js contains a valid key and descriptive app name. Sources: S1, S7.
9. Change the application's startup command to load New Relic first with node -r newrelic followed by the actual entry point. Success signal: The require flag precedes the application entry point. Sources: S1, S2, S9.
10. Start or restart the Node.js application with the modified command. Success signal: The app starts without New Relic configuration or module errors. Sources: S1.
11. Generate request traffic to the running application. Success signal: The application successfully handles one or more requests. Sources: S1, S8.
12. Wait a few minutes, then open New Relic APM and select the configured application name. Success signal: The application's APM page displays incoming performance data for the generated traffic. Sources: S1, S8, S9.

## Documented friction gates

- A New Relic account and email confirmation precede instrumentation; the Node guide says no credit card is required. Stage 1. Evidence state: documented requirement. Sources: S1, S3, S6.
- The general platform requires a telemetry-source and environment choice; guided install dynamically detects the environment and no source is canonical across New Relic. Stage 3. Evidence state: documented requirement. Sources: S2.
- The Node candidate requires a supported operating system, SHA-2, a compatible Node.js and agent-version pair, and supported module versions or manual API instrumentation. Stage 4. Evidence state: documented requirement. Sources: S1, S11.
- Almost all data ingestion requires an account license key, and role permissions determine which keys the developer can see or manage. Stage 7. Evidence state: documented requirement. Sources: S5.
- The agent must load before other application modules; startup methods differ for plain Node, Docker, Next.js, NestJS, and source-code fallback. Stage 9. Evidence state: documented requirement. Sources: S1, S9.
- The docs require generated traffic and a wait of a few minutes; the Node agent reports data once per minute. Stage 12. Evidence state: documented requirement. Sources: S1, S8, S9.
- No-data recovery can require inspecting newrelic_agent.log, correcting unsupported framework or module instrumentation, waiting one hour for a deleted app-name cache flush, restarting to reconnect, or waiting for collector connectivity to recover. Stage 12. Evidence state: documented requirement. Sources: S8, S9, S11.

## Unknowns

- Which telemetry source and environment should represent New Relic in the 100-platform comparison? Why unresolved: The general guide routes through dynamic guided install and lists many co-equal agents and data sources without establishing one platform-wide default; Node.js is a researched candidate only. Next evidence needed: Accounts are shared, but credential, installer, permissions, traffic generation, wait behavior, and success UI vary radically by source.
- What authenticated onboarding screens follow submission of the public Company Email and Name signup form for a new password-based account? Why unresolved: The official public signup enumerates its initial fields and social providers, and login documentation covers confirmation recovery, but the subsequent authenticated transaction is dynamic and not fully enumerated on the public page. Next evidence needed: Confirmation, organization, region, survey, or generated Add your data screens may add actions before the static Node.js route.
- What is the current supported UI method for recovering the full original account license key if it was not stored? Why unresolved: The current API-key documentation says stored keys are truncated and describes supported retrieval or replacement behavior, while older lifecycle wording changed UI copying; creating and securely capturing a new Ingest - License key is the deterministic route preserved here. Next evidence needed: Manual agent configuration cannot proceed without a full key.
