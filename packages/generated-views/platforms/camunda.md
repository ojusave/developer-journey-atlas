## Documented path

1. Open Get started with Camunda and select Run your first BPMN process. Success signal: The guide describes a local BPMN process that performs pre-flight, decision, destination, and mission-report steps. Sources: S1, S2.
2. Confirm that JDK 21 through 25 is installed, then download the Getting Started Package from Camunda Downloads. Success signal: The archive contains the local runtime, Modeler, and example-project files named by the guide. Sources: S1.
3. Unzip the package, change to its directory, and start Camunda 8 Run with the command for the operating system. Success signal: The packaged local services become available for Modeler and Operate. Sources: S1.
4. Open Camunda Modeler, select File and Open File, and open `camunda-8-get-started/1-rocket-launch/rocket-launch.bpmn`. Success signal: The model is open with the deployment and play controls available. Sources: S1.
5. Click the rocket icon, retain the preconfigured `c8run (local)` connection, and click Deploy. Success signal: Modeler completes the deployment through the local connection. Sources: S1.
6. Click Play and start a process instance with `missionName` set to `Odyssey` and `fuelLevel` set to 95. Success signal: The local engine accepts the instance and executes the supplied process. Sources: S1.
7. Open `http://localhost:8080/operate`, sign in with `demo` and `demo`, find Rocket Launch, and open the running instance. Success signal: The Rocket Launch instance page opens. Sources: S1.
8. Execute the process through its Countdown T-10 timer and mission logic. Success signal: Operate marks the instance completed successfully. Sources: S1.
9. Open the completed instance's Variables panel and inspect the generated result variables. Success signal: The completed successful instance and its Venus mission result establish the first BPMN process run. Sources: S1.
