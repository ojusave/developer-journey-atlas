**Developer goal:** Submit a sample video to Mux, wait for processing, and watch the streamed result.

**Meaningful success:** A publicly playable Mux asset reaches ready state and streams in the browser through its playback ID.

**Evidence boundary:** explicitly-named. It is the first rendered media result in Mux's official core onboarding guide.

## Documented path

1. Create an access token in the Mux dashboard. Success signal: Both credentials are displayed. Sources: S1.
2. POST a public video URL to the Mux assets endpoint using Basic authentication. Success signal: The API returns both identifiers. Sources: S1.
3. Wait until the asset's status becomes `ready`. Success signal: The retrieve-asset response reports `status: ready`. Sources: S1, S2.
4. Construct the playback URL from the returned Playback ID. Success signal: The generated URL contains the Playback ID. Sources: S1.
5. Open `https://stream.new/v/{PLAYBACK_ID}` in a browser. Success signal: The submitted video streams in the preview. Sources: S1.

## Documented friction gates

- Dashboard account access is required. Stage 1. Evidence state: documented requirement. Sources: S1.
- A Token ID and Secret with Video Read and Write permissions must be generated and stored. Stage 1. Evidence state: documented requirement. Sources: S1.

- Mux must download, transcode, and package the asset before status becomes ready. Stage 3. Evidence state: documented requirement. Sources: S1, S2.
