## Documented path

1. Sign up for a SendGrid account. Success signal: Account creation confirmation. Sources: S6.
2. Enable Two-factor authentication (2FA) on the account. Success signal: 2FA confirmation message. Sources: S6.
3. Create an API key with Mail Send > Full Access permissions. Success signal: API key creation confirmation. Sources: S6, S5.
4. Complete Domain Authentication to verify sender identity. Success signal: Domain authentication success notification. Sources: S6.
5. Set environment variable SENDGRID_API_KEY with the API key value. Success signal: Environment variable accessible to application. Sources: S6.
6. Write code to send an email using the SendGrid Mail Send API or SDK. Success signal: HTTP 202 Accepted response from SendGrid. Sources: S3, S4, S5, S6.
7. Run the code or curl command to send the email. Success signal: Email received in recipient inbox or HTTP 202 response. Sources: S5, S3, S4.

## Documented friction gates

- Must create a SendGrid account to access API and dashboard. Stage 1. Evidence state: documented requirement. Sources: S6.
- Two-factor authentication (2FA) is required to secure the account. Stage 2. Evidence state: documented requirement. Sources: S6.
- API key with Mail Send permissions must be created and stored securely. Stage 3. Evidence state: documented requirement. Sources: S6, S5.
- Domain Authentication is required to verify sender identity for production use. Stage 4. Evidence state: documented requirement. Sources: S6.
- Developer must have programming environment and optionally install SendGrid SDK. Stage 6. Evidence state: documented requirement. Sources: S6, S5.
