**Developer goal:** Create a Sinch account and follow Conversation API Getting Started until its SMS message request is sent.

**Meaningful success:** The CLI sends an SMS from the rented Sinch number to the developer's phone.

**Evidence boundary:** demonstrated-terminal-state. Receipt on the destination phone is the first end-to-end proof that account credentials, number rental, Conversation API configuration, and messaging work.

## Documented path

1. Open the official Sinch CLI Quickstart. Success signal: Prerequisites and commands load. Sources: S7.
2. Sign up for a free Sinch account at the official signup URL. Success signal: Project ID, access key ID, and key secret can be obtained. Sources: S3, S4, S5.
3. Install the Sinch CLI with `npm install -g @sinch/cli`. Success signal: The CLI can be invoked. Sources: S7.
4. Run `sinch auth login` and provide the project ID, key ID, and key secret. Success signal: Authentication completes without error. Sources: S7.
5. Search for an SMS-capable number and rent the selected number. Success signal: The CLI reports the rented number. Sources: S7.
6. Run `sinch conversation send <your_phone_number> "Hello from the Sinch CLI!"`. Success signal: The CLI prints a successful send result. Sources: S7.
7. Confirm the SMS arrives on the destination phone. Success signal: The phone receives `Hello from the Sinch CLI!`. Sources: S7.
