**Developer goal:** Start without dbt software or a project, follow the official dbt Core and DuckDB quickstart through either its Local or Web browser route, build the Jaffle Shop project, and observe the documented successful build summary.

**Meaningful success:** dbt Core seeds the sample DuckDB database, creates the Jaffle Shop view and table models, runs the data tests, and completes all 28 nodes without warnings, errors, or skips in the guide's demonstrated terminal.

**Evidence boundary:** demonstrated-terminal-state. The successful build is the guide's first proof that dbt Core, its DuckDB adapter, the provided project, seeds, models, and tests work together. ## Documented path

1. Open Quickstart for dbt Core using DuckDB and review its prerequisites and stated first build. Success signal: The Set up DuckDB for dbt Core section offers Local and Web browser tabs. Sources: S1.
2. Choose Local or Web browser before beginning setup. Success signal: The selected route can proceed to its documented `dbt build` command. Sources: S1.
3. Run `git clone https://github.com/dbt-labs/jaffle_shop_duckdb.git`. Success signal: The repository directory is available locally. Sources: S1.
4. Run `cd jaffle_shop_duckdb`. Success signal: `requirements.txt` and `dbt_project.yml` are available from the current directory. Sources: S1.
5. Create the virtual environment with `python3 -m venv venv` and activate it with `source venv/bin/activate`. Success signal: The shell prompt uses the `venv` environment for package installation. Sources: S1.
6. Run `python3 -m pip install --upgrade pip`, then `python3 -m pip install -r requirements.txt`, and activate the environment again as shown. Success signal: The environment can invoke `dbt`. Sources: S1.
7. Run `dbt build` from the project root. Success signal: The terminal prints `Completed successfully`. Sources: S1.
8. Read the final `dbt build` summary. Success signal: The documented terminal is `Done. PASS=28 WARN=0 ERROR=0 SKIP=0 TOTAL=28`. Sources: S1.
