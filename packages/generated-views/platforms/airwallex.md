**Developer goal:** Authenticate to the Airwallex sandbox API for the first time.

**Meaningful success:** The developer receives a sandbox bearer access token from the Authentication API.

**Evidence boundary:** explicitly-named. The general API overview explicitly defines obtaining an access token as what is needed to get started; product transactions and other endpoint calls follow this credential milestone.

## Documented path

1. Open the Airwallex Developer Tools API overview and follow the sandbox environment route. Success signal: The sandbox overview is open. Sources: S1, S2.
2. Open Get started in sandbox, enter business email, name, business country, and password, then select Create sandbox account. Success signal: The documentation says the sandbox account is ready to use instantly. Sources: S2, S6.
3. Provision and activate the sandbox account automatically. Success signal: Onboarding verification is SUCCESS and account status is ACTIVE. Sources: S2, S5.
4. Log in to the sandbox web app with the sandbox credentials. Success signal: The sandbox web app is available with test balances and product capabilities. Sources: S2.
5. Open the API overview and sandbox overview. Success signal: The official sandbox path is visible. Sources: S1, S2.
6. Enter business email, name, business country, and password, then select Create sandbox account. Success signal: The account is ready instantly. Sources: S2, S6.
7. Apply automatic sandbox verification, activation, and API settings. Success signal: Verification is SUCCESS and account status is ACTIVE. Sources: S5.
8. Log in to the sandbox web app. Success signal: The sandbox web app opens. Sources: S2.
9. Open Settings > Developer > API keys and select Generate next to the account. Success signal: The web app asks for the login password. Sources: S3.
10. Enter the login password, generate the admin API key, and copy the API key and Client ID. Success signal: The generated API key is displayed for copying. Sources: S3.
11. POST to `https://api-demo.airwallex.com/api/v1/authentication/login` with Content-Type, `x-client-id`, and `x-api-key` headers. Success signal: The request succeeds and returns JSON. Sources: S1, S2, S4.
12. Read the `token` and `expires_at` values from the successful response. Success signal: Both `token` and `expires_at` are present. Sources: S4.
