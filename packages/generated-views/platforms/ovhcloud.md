**Developer goal:** Create an OVHcloud account and active Public Cloud project, launch a first public Linux instance, and obtain an authenticated remote shell.

**Meaningful success:** OVHcloud reports the first Linux Public Cloud instance Enabled, and the developer has logged in through SSH as the image's preconfigured sudo user.

**Evidence boundary:** demonstrated-terminal-state. The official guide names the connection section `First login`, states that the instance is accessible immediately after creation, and moves password setup, user management, and additional SSH keys into Step 6 after the login. Application installation is also outside the getting-started boundary.

## Documented friction gates

- Country of tax residence, currency, and language selection precede credentials. The global and US project guides show that regional order and promotion details differ. Stage 2. Evidence state: documented requirement. Sources: S1, S2, S7.
- Email ownership is verified with a one-time code before account type and identity details. Stage 4. Evidence state: documented requirement. Sources: S1.
- Public Cloud contracts must be reviewed and accepted before the first project is created. Stage 6. Evidence state: documented requirement. Sources: S2.
- A first project without payment is placed in discovery mode, which looks browsable but cannot create an instance until activation. Stage 7. Evidence state: documented requirement. Sources: S2.
- A payment method is required to activate the project. The US order path additionally documents billing and contact confirmation, terms and privacy acceptance, a Pay action, and a $0.99 anti-fraud charge. Stage 8. Evidence state: documented requirement. Sources: S2, S7.
- Key-only Linux access adds local key generation, private-key protection, an optional passphrase, and later key matching. Stage 9. Evidence state: documented requirement. Sources: S3, S4.
- The model picker exposes eight families with different resources and use cases, while quotas and resize constraints can narrow the choice. Stage 14. Evidence state: documented requirement. Sources: S3, S5.
- Automated backups are enabled by default and require a 7- or 14-day rotation decision before launch. Stage 16. Evidence state: documented requirement. Sources: S3.
- Public, Private, and Local Private modes have different IP, gateway, Floating IP, and vRack requirements. The primary path requires an explicit public-connectivity choice. Stage 17. Evidence state: documented requirement. Sources: S3.
- Delivery can take a few minutes before the status becomes Enabled; the user may need to refresh the table. Stage 20. Evidence state: documented requirement. Sources: S3.
- The preconfigured sudo username varies by image and must be retrieved from the Dashboard rather than assumed. Stage 21. Evidence state: documented requirement. Sources: S3.
- A lost, wrong, or omitted SSH key requires verification or rescue-mode replacement. Linux VNC is not a zero-setup fallback because a password must first be configured. Stage 22. Evidence state: documented requirement. Sources: S3, S4.

## Unknowns

- Which payment instruments, authorization amount, and anti-fraud checks apply to each regional OVHcloud account? Why unresolved: The global project guide requires payment registration but does not enumerate universal methods or challenges, while the US guide documents a region-specific $0.99 transaction. Next evidence needed: The number of billing steps and time before project activation can vary materially.
- Which model, Linux image, and location should be treated as the universal tutorial default? Why unresolved: The getting-started guide intentionally asks the user to choose and provides category guidance without naming one universal combination. Next evidence needed: The exact cost, username, performance, quota, and availability differ across otherwise valid first paths.
- What live price, if any, is shown for the default automated backup in each region and model combination? Why unresolved: The guide documents that automated backup is enabled by default and requires a 7- or 14-day rotation, but it does not publish a universal backup price in the cited section. Next evidence needed: The total first-success order cost cannot be calculated from the getting-started guide alone.
- What exact shell prompt or command output should be captured after successful SSH authentication? Next evidence needed: The connection boundary is observable as an opened remote command line but lacks one vendor-specified verification command such as `whoami`.
