## Documented friction gates

- An Algolia account and JavaScript-enabled browser are required. Stage 3. Evidence state: documented requirement. Sources: S2, S3.
- Work Email signup requires acceptance of the Terms of Service, Build Plan Terms, Service Limits, and privacy notice. Stage 5. Evidence state: documented requirement. Sources: S3.
- The quickstart requires an application but does not establish whether signup creates one. Manual creation adds plan, billing-if-needed, and cluster-region choices. Stage 6. Evidence state: documented requirement. Sources: S2, S4, S7.
- The route requires Node.js 20.19 or later and five npm packages. Stage 8. Evidence state: documented requirement. Sources: S2.
- Three dashboard values must be copied and separated correctly between indexing and frontend use. The Write API key must not be committed or exposed. Stage 9. Evidence state: documented requirement. Sources: S2, S5.
- The developer must populate a TypeScript indexing script, then replace a React component and a large CSS file with supplied source. Stage 12. Evidence state: documented requirement. Sources: S2.
- Algolia indexing and settings operations are asynchronous; the official script waits for both before reporting success. Stage 14. Evidence state: documented requirement. Sources: S2.
- The indexing script reports missing Application ID, missing Write API key, failed sample-dataset response, and caught indexing failures, but the guide provides no procedural recovery beyond correcting the cause and rerunning. Stage 14. Evidence state: documented requirement. Sources: S2.
