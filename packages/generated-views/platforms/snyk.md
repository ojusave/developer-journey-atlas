## Documented path

1. Open Snyk's general getting-started page and identify the target system and Project type to scan. Success signal: No shared platform success signal exists at this choice point. Sources: S1.
2. Choose guided or manual SCM integration, or the local CLI route. Success signal: The selected route can terminate at a documented scan report. Sources: S1, S2, S3, S4.
3. Create or sign in to the correct Snyk account and Organization, or complete the company's SSO invitation. Success signal: The Snyk Web UI opens in the intended account context. Sources: S1, S4.
4. Install the Snyk CLI using one documented method and make the `snyk` command available. Success signal: The `snyk` command can be invoked from the terminal. Sources: S2, S3.
5. If the account is outside `SNYK-US-01`, run `snyk config environment <ENVIRONMENT>` with its documented regional value before authentication. Success signal: The CLI stores the correct regional environment. Sources: S4.
6. Run `snyk auth` to start the recommended OAuth authentication flow. Success signal: The Snyk authentication page opens or its URL is provided. Sources: S2, S4.
7. Log in if prompted, grant the Snyk CLI application access, complete browser confirmation, and return to the terminal. Success signal: The terminal prints `Your account has been authenticated` and regains control. Sources: S4.
8. Run `snyk test ionic` to perform the documented vulnerability quick test against the public Ionic npm package. Success signal: The CLI runs the Ionic vulnerability test instead of reporting installation or authentication failure. Sources: S2.
9. Inspect the terminal test report. Success signal: The terminal displays the issue report and its remediation guidance. Sources: S2.
