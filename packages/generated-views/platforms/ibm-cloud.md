## Documented path

1. Open the general IBM Cloud onboarding checklist and begin with Explore the platform. Success signal: The developer reaches the documented list of IBM Cloud working options. Sources: S1.
2. Open IBM Cloud account registration and create the account with an email address or Google credentials. Success signal: The new account can proceed to its first login. Sources: S1.
3. On first login, provide a credit card or subscription code to complete account setup. Success signal: The IBM Cloud console opens for the account owner. Sources: S1.
4. Open the Code Engine console and select Start creating. Success signal: The page offers project and workload choices. Sources: S4.
5. Select Create serverless project. Success signal: Location, name, and resource-group controls are available. Sources: S4, S7.
6. Choose the Code Engine project location. Success signal: The location is accepted. Sources: S7.
7. Enter a unique project name, choose a resource group, and click Create. Success signal: The project is listed and can be selected for workload creation. Sources: S7.
8. Return to Start creating, select the new project, and choose Application. Success signal: Application settings are displayed. Sources: S4.
9. Enter a project-unique application name such as myapp. Success signal: The name passes validation. Sources: S4.
10. Select Use an existing container image and enter icr.io/codeengine/helloworld. Success signal: The image reference is accepted. Sources: S4.
11. Click Create to deploy the application. Success signal: The application page begins showing deployment status. Sources: S4.
12. Wait for the application status to change to Ready. Success signal: Status is Ready and Test application is available. Sources: S4.
13. Click Test application and then Send request. Success signal: A response is returned in the test pane. Sources: S4.
14. Verify that the response begins with Hello World and includes Code Engine environment values. Success signal: The Hello World response is visible. Sources: S4.

## Documented friction gates

- IBM Cloud requires the developer to choose both a product or service from a catalog of hundreds and one of several working interfaces before a single first-mile path exists. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- Interactive candidate paths require IBM Cloud account access. The documented account route differs for a new account owner and a user invited to an existing account. Stage 2. Evidence state: documented requirement. Sources: S1, S3, S6.
- For a developer creating a new stand-alone account as its owner, the onboarding checklist says the first login requires a credit card or subscription code to complete account setup. Invited users are explicitly documented as not needing to create their own account. Stage 2. Evidence state: documented requirement. Sources: S1.
- On the Cloud Shell branch, account-level feature settings can disable Cloud Shell features, and an account administrator can grant user-level access. Stage 2. Evidence state: documented requirement. Sources: S3.
