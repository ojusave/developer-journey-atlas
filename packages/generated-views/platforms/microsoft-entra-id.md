## Documented path

1. Open the Microsoft identity platform documentation. Success signal: The available identity outcomes and application types are visible. Sources: S1.
2. Obtain a Microsoft Entra tenant and an account with application-registration access. Success signal: Microsoft Entra admin center access and the target tenant are available. Sources: S2.
3. Choose the first identity outcome to implement. Success signal: The selected family matches the application goal. Sources: S1, S5.
4. Choose the application type and OAuth client model. Success signal: The app type matches the software being integrated. Sources: S1, S3, S4.
5. Sign in to a Microsoft Entra tenant and confirm that the .NET 9 SDK is installed locally. Success signal: The Entra admin center or Azure portal is accessible and dotnet is available. Sources: S2.
6. Open Microsoft Entra ID, go to App registrations, and select New registration. Success signal: The registration form is visible. Sources: S2.
7. Enter a display name, select the supported account type, and add a Web redirect URI of https://localhost:5001/signin-oidc. Success signal: The form contains the Web callback exactly as documented. Sources: S2.
8. Register the application and copy its Application client ID and Directory tenant ID. Success signal: The app Overview page shows both identifiers and they are copied. Sources: S2.
9. Scaffold the sample with dotnet new webapp --auth SingleOrg --name MyWebApp, then enter the MyWebApp directory. Success signal: The MyWebApp directory and project files are created. Sources: S2.
10. Update appsettings.json with the tenant ID, client ID, and callback path from the registration. Success signal: The Microsoft identity configuration matches the Entra registration. Sources: S2.
11. Run the application with dotnet run. Success signal: The command reports that the app is listening locally. Sources: S2.
12. Open https://localhost:5001 and select Sign in. Success signal: The Microsoft sign-in prompt appears, which the quick check treats as correct basic configuration. Sources: S2.
13. Complete Microsoft sign-in and confirm the browser returns to the local application as an authenticated user. Success signal: The user is redirected back to the app after sign-in without an Entra error. Sources: S2.
