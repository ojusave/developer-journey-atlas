## Documented friction gates

- The developer-account form requires personal identity, company, country or region, and build-intent fields. Stage 2. Evidence state: documented requirement. Sources: S3.
- Get Started accepts Docusign's Terms and Conditions and Privacy Policy. Stage 2. Evidence state: documented requirement. Sources: S3.
- Verification, sandbox provisioning, and authenticated Quickstart access are hidden after form submission, and Quickstart requires the account owner or admin because it creates an app and integration key. Stage 3. Evidence state: documented requirement. Sources: S1, S3, S4.
- The generator requires an app name, language, and project-type choice; the overview identifies Multiple code examples as the main project and documents its first-run default. Stage 3. Evidence state: documented requirement. Sources: S1, S4.
- Quickstart creates and preconfigures the selected route's app, integration key, account settings, and authentication code; manual credential setup applies only to the unconfigured launcher branch. Stage 3. Evidence state: documented requirement. Sources: S1, S2, S4.
- The selected main project first authenticates interactively with Authorization Code Grant; JWT alternatives can require first-time impersonation consent. Stage 3. Evidence state: documented requirement. Sources: S2, S4.
- The selected Node.js Quickstart requires ZIP extraction, Node.js 8.10 or later, npm 5 or later, `npm install`, `npm start`, and a local browser session. Stage 3. Evidence state: documented requirement. Sources: S2, S4.
