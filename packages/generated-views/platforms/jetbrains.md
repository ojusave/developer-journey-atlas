**Developer goal:** Choose a JetBrains developer product and reach the first terminal state that its official getting-started material defines; for the retained candidate, run a generated IntelliJ Platform plugin locally.

**Meaningful success:** The wizard-generated Gradle plugin project starts the target IntelliJ Platform IDE in a standalone development instance with the plugin loaded.

**Evidence boundary:** explicitly-named. Local execution is the earliest documented proof that the generated plugin project builds and can be loaded by its target IDE. Adding visible actions, tests, compatibility verification, signing, packaging, and Marketplace publication are later work.

## Documented path

1. Open the JetBrains product catalog and choose the product surface whose first mile is being measured. Success signal: A product and its own documentation can be selected. Sources: S4.
2. For the retained candidate, choose IntelliJ Platform plugin development and open Developing a Plugin. Success signal: Creating a Plugin Gradle Project is listed as the first workflow step. Sources: S1, S4.
3. Open Developing a Plugin and select the recommended Gradle-based New Project Wizard workflow. Success signal: Creating a Plugin Gradle Project is listed as the first workflow step. Sources: S1.
4. Install the JetBrains Toolbox App using the operating-system-specific instructions, launch it, and select IntelliJ IDEA to install. Success signal: IntelliJ IDEA appears as an installed Toolbox product and launches. Sources: S3.
5. In IntelliJ IDEA, install and enable Plugin DevKit from JetBrains Marketplace and verify Gradle support is enabled. Success signal: IDE Plugin is available as a New Project type. Sources: S1, S2.
6. Open File > New > Project, select IDE Plugin, and configure the plugin project. Success signal: The project opens and contains the documented generated structure and run configuration. Sources: S2.
7. Execute Run IDE with Plugin from Run > Run or the Gradle Run Configurations node. Success signal: The standalone IDE opens without a build or launch error. Sources: S2.

## Documented friction gates

- The installer and exact Toolbox procedure differ by operating system and, on macOS and Linux, processor architecture. Stage 2. Evidence state: documented requirement. Sources: S3.
- The path requires Toolbox and IntelliJ IDEA desktop installation with permission to run downloaded software. Stage 2. Evidence state: documented requirement. Sources: S3.
- Inside the selected candidate, Plugin DevKit is no longer bundled and must be installed from JetBrains Marketplace. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- Inside the selected candidate, target platform, Gradle plugin generation, and JDK versions must be compatible. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- Inside the selected candidate, the generated wrapper can download its specified Gradle version automatically. Stage 2. Evidence state: documented requirement. Sources: S2.
