## Documented path

1. Open Install TimescaleDB from a Docker container. Success signal: The Install and configure TimescaleDB on Postgres section is reached. Sources: S9.
2. Run the Docker commands published in the page's Docker tab to start the latest Tiger Data TimescaleDB container with the documented Postgres password and port mapping. Success signal: Docker reports the container as running. Sources: S9.
3. Connect to the local database with the psql command published by the Docker guide. Success signal: An interactive psql prompt is displayed. Sources: S9.

## Documented friction gates

- The unauthenticated static signup response contains only the JavaScript application shell and noscript message, while official prose says only to add details or sign up. Stage 2. Evidence state: documented requirement. Sources: S1, S2, S4.
- A new user can choose the Free Plan or a 30-day performance trial, which changes limits, resources, and lifecycle. Stage 2. Evidence state: documented requirement. Sources: S2.
- The current service-creation flow has three steps and a persistent price display, but public docs do not enumerate every required field and default in those steps. Stage 2. Evidence state: documented requirement. Sources: S3.
- The generated config includes passwords and connection information and must be stored securely. Stage 2. Evidence state: documented requirement. Sources: S1.
- Warnings appear as the service approaches 750 MB; at the limit the service becomes read-only, with documented options to fork, delete data, or upgrade and convert. Stage 2. Evidence state: documented requirement. Sources: S2.
- Free services omit connection pooling, compute resizing, dedicated support, high availability, advanced security, and enterprise features. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
