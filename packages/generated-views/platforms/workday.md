2. Choose direct API use, an Extend app, or an Integration app. Success signal: The surface matches the use case. Sources: S1, S7.
3. Obtain Developer Site and tenant access for the selected surface. Success signal: The developer can open the relevant console, App Builder, API Explorer, or tenant. Sources: S4, S6, S7, S8.
4. Choose the API service, version, endpoint or app tutorial, and acting identity. Success signal: Service, version, operation, identity, and expected response are known. Sources: S2, S3, S5.
5. Follow the selected guide's request, build, deploy, or orchestration steps and verify its terminal result. Success signal: The endpoint returns authorized data or the selected app/integration reaches its documented final state. Sources: S2, S4, S7.
6. Confirm that the company has eligible Enterprise Developer Site access, a provisioned Workday Cloud Platform Development tenant, and a company administrator who can invite the developer. Success signal: The administrator can open company user management and the Development tenant is available for assignment. Sources: S4, S6, S8.
7. Open the Developer Site Console, go to Users, and select Add a User. Success signal: The Add a User fields are visible. Sources: S6.
8. Enter the developer's first name, last name, company email, and company role, then select Add User. Success signal: The new user is added and Workday initiates the setup email. Sources: S6.
9. Deliver the Workday Developer Site account-setup email to the invited developer. Success signal: The invited developer receives the setup link. Sources: S6.
10. Open the setup link and complete the Developer Site access setup described by the invitation. Success signal: The setup link completes the user's Developer Site access setup. Sources: S6.
11. Sign in to the Workday Developer Site and open App Builder. Success signal: The App Builder workspace opens. Sources: S4, S6.
12. Open the tenant connection control represented by the cylinder and lightning icon. Success signal: API Explorers and App Builder connection targets are visible. Sources: S4.
13. Select App Builder as the connection target. Success signal: The dialog exposes the available Development tenant choice. Sources: S4.
14. Select the Development tenant, choose Bypass SSO only if required, and confirm with OK. Success signal: The connection dialog closes after OK and App Builder evaluates the connection. Sources: S4.
15. Verify that App Builder shows the Development tenant connection with a green status dot. Success signal: The connection indicator is green. Sources: S4.
