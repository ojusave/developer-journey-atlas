## Documented friction gates

- Signup begins with an account-type decision that changes required profile and phone fields. Stage 2. Evidence state: documented requirement. Sources: S1.
- Direct email signup requires opening a validation link before account creation can finish. Stage 4. Evidence state: documented requirement. Sources: S1.
- Phone verification is mandatory outside the Personal project account type and adds SMS delivery and code-entry dependency. Stage 5. Evidence state: documented requirement. Sources: S1.
- A valid card is mandatory before the first resource order, virtual and prepaid cards are rejected, a reimbursed €1 authorization is made, and 3D Secure can add an external bank step. Stage 7. Evidence state: documented requirement. Sources: S1, S2, S3.
- Passwordless access requires local key generation, private-key storage, and an optional passphrase decision before any Instance configuration. Stage 8. Evidence state: documented requirement. Sources: S5.
- SSH keys are Project-scoped, so selecting or uploading to the wrong Project creates a later authentication failure. Stage 9. Evidence state: documented requirement. Sources: S5.
- Instance range and size decisions expose price, performance, memory, storage, bandwidth, zone availability, and capacity tradeoffs before first use. Stage 11. Evidence state: documented requirement. Sources: S6, S8.
- The system volume must be at least 10 GB for a standard OS, and multiple volumes with operating systems can produce inconsistent boot outcomes. Stage 14. Evidence state: documented requirement. Sources: S6.
- For public-key rejection, Scaleway directs users to verify the Project key and either reboot the Instance or refresh keys with `scw-fetch-ssh-keys --upgrade` from an existing session. Stage 19. Evidence state: documented requirement. Sources: S7.
