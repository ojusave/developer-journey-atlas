**Developer goal:** Create a free InfluxDB Cloud Serverless account and use the official browser tutorial to write its published home-sensor line protocol sample successfully to the automatically created default bucket.

**Meaningful success:** The developer writes InfluxData's published home-sensor line protocol sample to the automatically created default bucket through the InfluxDB Cloud Serverless UI.

**Evidence boundary:** explicitly-named. Writing data is part 2 of the official three-part get-started tutorial and supplies an explicit browser success confirmation. The separate SQL query journey is part 3 and depends on the data already written, so it is after the first successful platform operation.

## Documented path

1. Open the official InfluxDB Cloud Serverless signup guide and follow its InfluxDB Cloud link under Subscribe through InfluxData. Success signal: The email or social signup choices are available. Sources: S1.
2. Choose email signup, enter name, email address, and password, then select Create Account. Success signal: The verification email is sent as the next documented signup state. Sources: S1.
3. Open the InfluxDB email and select Verify Your Email. Success signal: The provider and region setup becomes the next documented state. Sources: S1.
4. Choose Amazon Web Services and select US East (Virginia) for the Cloud Serverless instance. Success signal: The form can continue to company and agreement inputs. Sources: S1.
5. Enter the company name. Success signal: The account setup has its required company identity. Sources: S1.
6. Review the Cloud Services Subscription Agreement and Global Data Processing Agreement, then select the statement affirming both. Success signal: Continue becomes the documented next action. Sources: S1.
7. Select Continue, then choose Keep to remain on the Free Plan. Success signal: A default organization and bucket, both named from the email address, are created automatically and the Cloud UI opens. Sources: S1.
8. In Load Data, then Buckets, select Add Data on the automatically created default bucket and choose Line Protocol. Success signal: Enter Manually is available. Sources: S1, S3.
9. Select Enter Manually and set the Precision menu to Seconds to match the published timestamps. Success signal: The line-protocol text field is ready for the sample. Sources: S3.
10. Copy the official home-sensor line protocol sample and paste it into the line-protocol field. Success signal: Write Data is ready to submit the records. Sources: S3.
11. Select Write Data. Success signal: The UI confirms that the data has been written successfully. Sources: S3.

## Documented friction gates

- Email signup requires opening a verification message before Cloud setup can continue. Stage 2. Evidence state: documented requirement. Sources: S1.
- The documented direct subscription requires choosing AWS and either US East (Virginia) or EU Frankfurt. Stage 4. Evidence state: documented requirement. Sources: S1.
- The direct route requires affirmative acceptance of two agreements. Stage 6. Evidence state: documented requirement. Sources: S1.
- Keep, Upgrade Now, and Contact Sales lead to materially different free, paid, and human-assisted paths. Stage 7. Evidence state: documented requirement. Sources: S1.
- The sample timestamps use seconds, so the UI precision must be changed to Seconds before writing. Stage 9. Evidence state: documented requirement. Sources: S3.
- Line protocol is whitespace-sensitive and requires its measurement, tag, field, and timestamp structure to remain intact. Stage 10. Evidence state: documented requirement. Sources: S3.
- If a Free Plan write or query rate is exceeded, the API returns HTTP 429 and Retry-After; wait for the fixed five-minute window or upgrade to raise organization limits. Stage 11. Evidence state: documented requirement. Sources: S5.
