**Developer goal:** Install ggshield, obtain GitGuardian access, and scan a local Git repository for hardcoded secrets until the CLI reports a completed scan.

**Meaningful success:** ggshield has authenticated to a GitGuardian workspace and completed a secret scan of the selected local repository.

**Evidence boundary:** explicitly-named. The Getting started page explicitly labels this as the first content scan, and the CLI reference defines its completed-scan exit codes.

## Documented path

1. Open the ggshield Getting started page and select the recommended Linux or macOS install-script route. Success signal: The page identifies the install script as the recommended quickest setup. Sources: S1.
2. Run the published install script. Success signal: The CLI begins the browser authentication route. Sources: S1.
3. Open GitGuardian browser authentication and instruct the developer to log in or create an account. Success signal: The browser offers an account-access route. Sources: S1, S2.
4. Sign up for or sign in to GitGuardian and select a workspace. Success signal: The browser authentication flow can authorize ggshield. Sources: S1, S2, S3.
5. Authorize ggshield for the workspace. Success signal: The browser flow completes and returns control to an authenticated CLI. Sources: S1.
6. Scan the local repository. Success signal: The command exits 0 when no issues are detected or exits 1 when issues are detected; both indicate a completed scan. Sources: S1, S4, S5.

## Documented friction gates

- The selected route downloads and executes GitGuardian's published installer before authentication. Stage 2. Evidence state: documented requirement. Sources: S1.
- All ggshield scans use the GitGuardian API and require a workspace plus API token. Stage 3. Evidence state: documented requirement. Sources: S1.
- A new user must choose GitHub, email, or organization SSO account access. Stage 4. Evidence state: documented requirement. Sources: S2.
- The GitHub route requires GitGuardian App installation and a choice of all or selected repositories. Stage 4. Evidence state: documented requirement. Sources: S2.
- The email route requires receiving and following a GitGuardian email. Stage 4. Evidence state: documented requirement. Sources: S2.
