**Developer goal:** Start without a Dropbox account, create an app in the Developer Console for own-account testing, select minimal content access and the `files.metadata.read` permission, generate an access token, install the JavaScript SDK, and list the root folder from Node.js.

**Meaningful success:** The Node.js `filesListFolder({ path: '' })` call resolves and the example logs the returned `result.entries` array for the account's root folder.

**Evidence boundary:** demonstrated-terminal-state. The SDK Basic example is the first authenticated API result. OAuth for other users and additional file operations are later work.

## Documented path

1. Open the official Dropbox JavaScript SDK page and identify the Node.js root-list example. Success signal: The Node.js snippet and its `result.entries` log are visible. Sources: S1.
2. Review the OAuth guide's own-account generated-token route and app-permission requirements. Success signal: Testing with a generated token is explicitly identified as a quick own-account route. Sources: S2.
3. Sign up for or sign in to Dropbox. Success signal: The Developer Console is accessible. Sources: S3.
4. Create a Dropbox API app with the least content access needed. Success signal: The app settings and OAuth 2 section are available. Sources: S2, S4, S5.
5. Generate an own-account access token from the app's OAuth 2 settings. Success signal: The generated token is displayed. Sources: S2.
6. Install the Dropbox JavaScript SDK. Success signal: npm completes successfully. Sources: S1.
7. Run the Basic root-list request using the token. Success signal: `result.entries` is printed, including an empty array for an empty accessible root. Sources: S1, S5.

## Documented friction gates

- A Dropbox account is required, but the signup page's fields and completion sequence are client-rendered and unavailable publicly. Stage 3. Evidence state: documented requirement. Sources: S3.
- The app-creation URL redirects to login before any Developer Console fields or token controls are available. Stage 4. Evidence state: documented requirement. Sources: S4.
- App Folder and Full Dropbox are peer content-access choices with different data reach. Stage 4. Evidence state: documented requirement. Sources: S2.
- The app must enable the endpoint's `files.metadata.read` scope, and Dropbox recommends least privilege. Stage 4. Evidence state: documented requirement. Sources: S2, S5.
- The generated access token is limited to own-account testing and must not be hard-coded in public source. Stage 4. Evidence state: documented requirement. Sources: S1, S2.
- The selected SDK path requires Node.js 22 or newer, npm installation, and an environment variable. Stage 4. Evidence state: documented requirement. Sources: S1.
