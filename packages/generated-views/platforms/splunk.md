**Developer goal:** Create a Splunk account, choose a trial platform, upload the official tutorial data, and confirm that Splunk indexed it as events.

**Meaningful success:** Splunk Cloud indexes an uploaded sample log and the Search app displays it as time-stamped events.

**Evidence boundary:** explicitly-named. Visible indexed events are the first proof that the Cloud deployment, upload, parsing, indexing, and search path work.

## Documented path

1. Open the Search Tutorial prerequisites and review the required account, trial-platform choice, system requirements, and tutorial files. Success signal: The page shows Create a splunk.com account, Choose a platform, Download the tutorial data files, and Access the Trial version sections. Sources: S1.
2. Create a Splunk.com account at the official signup URL. Success signal: The Cloud trial can be started. Sources: S1, S2.
3. Start the Splunk Cloud Platform free trial and wait for provisioning. Success signal: The trial-ready confirmation appears. Sources: S1, S3.
4. Click View My Instance, accept the Cloud terms, and log in with the provided URL and credentials. Success signal: The Splunk Web UI is interactive. Sources: S1, S5.
5. Choose a standard local log or text file for the Cloud Quick Start. Success signal: The file can be selected in the browser. Sources: S5.
6. Open Settings > Add Data, choose Upload, select the file, configure its source type and target index, review, and click Submit. Success signal: Splunk Web displays Success. Sources: S5.
7. Click Start searching and verify the uploaded data appears as time-stamped events. Success signal: Events from the uploaded file are visible. Sources: S5.
