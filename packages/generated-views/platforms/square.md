**Developer goal:** Create a Square developer application, make the documented first Sandbox API call, and verify the resulting payment.

**Meaningful success:** A Square developer application creates a completed Sandbox payment through API Explorer, exposes its request and response in API Logs, and shows the transaction in the Sandbox Square Dashboard.

**Evidence boundary:** explicitly-named. It is the final, seller-visible verification explicitly required by the official four-step platform journey.

## Documented path

1. Create a Square account or sign in, then open the Developer Console. Success signal: The Developer Console opens. Sources: S2.
2. Create a developer application and supply its name and optional preferences. Success signal: The application page appears in Developer Console. Sources: S2, S3.
3. Open API Explorer from the application's first-test-payment banner and select Run Request on the prefilled CreatePayment request. Success signal: The response contains a Payment object with `status` equal to `COMPLETED`. Sources: S3.
4. From the API Explorer response, choose View Logs, select the API call, and inspect its Request and Response tabs. Success signal: The matching call and both HTTP payload views are visible. Sources: S4.
5. Return to Developer Console, open Sandbox test accounts, and choose Square Dashboard for the Default Test Account. Success signal: The seller-style Sandbox dashboard opens. Sources: S5.
6. Open Transactions and select the payment created in API Explorer. Success signal: The matching payment appears and opens in Transactions. Sources: S5.

## Documented friction gates

- A Square account and required registration fields and agreements precede Developer Console access. Stage 1. Evidence state: documented requirement. Sources: S2.
- A developer application is required even though API Explorer prefills the Sandbox token and request. Stage 2. Evidence state: documented requirement. Sources: S2, S3.
- The first API call succeeds only if CreatePayment returns a Payment object with `COMPLETED` status. Stage 3. Evidence state: documented requirement. Sources: S3.
- Final confirmation occurs in a separate seller-facing Sandbox dashboard and can be obscured by its date filter. Stage 6. Evidence state: documented requirement. Sources: S5.
