**Developer goal:** Start using Google Cloud after choosing the platform but before choosing a product or job-function journey.

**Meaningful success:** Create a project with billing, deploy the Cloud Run sample container, and open its public service URL to see the container's response.

**Evidence boundary:** demonstrated-terminal-state. The research rules prohibit choosing among co-equal official routes. The record therefore stops at the documented platform router and records the missing product or job choice.

## Documented path

1. Open Get started with Google Cloud. Success signal: The account actions and job-function options are visible. Sources: S1.
2. Sign in to a Google Cloud account or create one if new to Google Cloud. Success signal: The project selector is available. Sources: S4, S5.
3. If using the optional Free Trial route for billing, start the trial, provide the requested name, address, and payment method for identity verification, and complete any work-account one-time-passcode email challenge. Success signal: For the optional Free Trial route, the billing account is active and the Welcome credit is available. Sources: S4, S5.
4. On the project selector page, select an existing project or create a new Google Cloud project. Success signal: The intended project is active in the console selector. Sources: S4.
5. Verify that billing is enabled for the selected project. Success signal: The project is linked to an active billing account. Sources: S4, S5.
6. Verify the developer has Cloud Run Admin on the service, Service Account User on the service identity, and Logs Viewer on the project. Success signal: All three required roles are granted directly or through equivalent custom or predefined roles. Sources: S4.
7. Open the Cloud Run page in the Google Cloud console. Success signal: The Cloud Run navigation and Services view are available. Sources: S4.
8. Select Services, then select Deploy container. Success signal: The deployment-source and service settings are visible. Sources: S4.
9. Ensure Deploy one revision from an existing container image is selected. Success signal: The Container image URL control is enabled. Sources: S4.
10. In Container image URL, select Test with a sample container. Success signal: The sample image URL populates the field. Sources: S4.
11. Enter a service name or keep the default. Success signal: The service-name field is valid. Sources: S4.
12. Keep the default europe-west1 region or select the intended Cloud Run region. Success signal: A supported region is selected. Sources: S4.
13. In Authentication, select Allow public access. Success signal: Allow public access is selected. Sources: S4.
14. Select Create and wait for Cloud Run to deploy the container. Success signal: Deployment completes and a URL appears beside URL:. Sources: S4.
15. Copy the displayed service URL, paste it into a browser, and open it. Success signal: The browser displays the response from the deployed Cloud Run sample container. Sources: S4.

## Documented friction gates

- The official platform get-started page begins with creating a Google Cloud account or going to the console. Stage 1. Evidence state: documented requirement. Sources: S1.
- A job function, product, or workload must be selected before the journey and terminal success can be defined. Stage 2. Evidence state: documented requirement. Sources: S1, S2, S3, S4.
- The optional Free Trial asks for name, address, and a supported payment method for identity verification. A pending authorization can remain for 1 to 14 business days, but Google states it is not a charge and does not bill unless the user manually upgrades. Stage 1. Evidence state: documented requirement. Sources: S5.
- Free Trial signup can require a work-account email one-time passcode, payment-method troubleshooting, a 24-hour retry wait after persistent signup errors, and a 1 to 14 business-day pending authorization. Stage 1. Evidence state: documented requirement. Sources: S5.
