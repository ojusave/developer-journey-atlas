## Documented friction gates

- Direct signup requires at least 12 characters and four character classes. Stage 3. Evidence state: documented requirement. Sources: S2.
- The primary route leaves the browser to retrieve a confirmation code, and the guide does not publish delivery or resend timing. Stage 4. Evidence state: documented requirement. Sources: S2.
- The GitHub branch requires the primary email to be both verified and public, adding a GitHub profile-setting change for developers who keep addresses private. Stage 2. Evidence state: documented requirement. Sources: S2.
- The recommended Guided Setup combines database and App Services onboarding, while manual Getting Started is the direct route to the general database query. The user must choose based on workload intent. Stage 6. Evidence state: documented requirement. Sources: S2.
- Capella allows only one free tier operational cluster per organization at any time. Stage 8. Evidence state: documented requirement. Sources: S2, S5.
- The free path still asks the developer to choose provider, region, and CIDR without naming a universal default combination. Stage 10. Evidence state: documented requirement. Sources: S2.
- The free-tier guide does not document an exact readiness status or deployment duration before Data Tools can query travel-sample. Stage 14. Evidence state: documented requirement. Sources: S2, S3.
- After more than 72 hours without activity, Couchbase turns off the free cluster and linked App Services while preserving data and state. An authorized user must select Turn On Cluster before continuing. Stage 15. Evidence state: documented requirement. Sources: S2.
- The tutorial specifies the successful JSON result but does not provide a dedicated error-recovery sequence for a failed first query. Stage 17. Evidence state: documented requirement. Sources: S3.
