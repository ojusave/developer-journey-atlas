**Developer goal:** Use the public Dynatrace Playground to explore and interact with live dashboards, notebooks, and sample observability data without installing software.

**Meaningful success:** The developer opens the Dynatrace Playground and interacts with a live dashboard or notebook backed by sample data.

**Evidence boundary:** explicitly-named. This is the first documented usable Dynatrace state and requires no software installation or data ingestion. Trial ingestion of the developer's own environment is a later alternative.

## Documented path

1. Open Discover Dynatrace and select Playground. Success signal: `Sign up for Playground` is available. Sources: S2, S5.
2. Sign up for Playground or sign in to an existing Dynatrace account. Success signal: Playground can be opened directly or from the environment switcher. Sources: S5, S6.
3. Open a Playground use case, live dashboard, or notebook. Success signal: A dashboard or notebook loads with sample data and responds to interaction. Sources: S5.
4. Verify that sample data can be explored without installing OneAgent or ingesting OpenTelemetry data. Success signal: Sample data is visible and interactive. Sources: S2, S5.

## Documented friction gates

- The public trial endpoint does not expose the required identity, agreement, account, tenant, or environment transition. Stage 2. Evidence state: documented requirement. Sources: S2, S4.
- The official general guide leaves OneAgent versus OpenTelemetry ingestion to the developer. Stage 2. Evidence state: documented requirement. Sources: S2.
- OneAgent requires administrator access to a clean supported host. Stage 2. Evidence state: documented requirement. Sources: S3.
- OneAgent setup generates an API token that cannot be accessed again after the generation screen. Stage 2. Evidence state: documented requirement. Sources: S3.
- Running processes must be restarted because OneAgent cannot inject itself into them while they are already running. Stage 2. Evidence state: documented requirement. Sources: S3.
