## Documented path

1. Open the SmartBear developer portal and review the available API and MCP product surfaces. Success signal: Each product links to its own documentation. Sources: S1.
2. Choose the SmartBear product that matches the intended job and follow that product's account, setup, and first-success path. Success signal: The chosen product's getting-started documentation is open. Sources: S1, S2.
3. Open Swagger Studio About, Creating an Account, and Basics of Swagger Studio. Success signal: The About page says to start by signing up and creating the first API. Sources: S2, S3, S4, S5.
4. Open the Swagger Studio signup page, choose email and password, and enter both when prompted. Success signal: The email-verification step is reached. Sources: S3, S5.
5. Complete the mandatory email verification, then access the SmartBear ID account. Success signal: The SmartBear ID required to use Swagger Studio is available. Sources: S3, S5.
6. Open Swagger Studio and arrive at My APIs. Success signal: APIs the user can access are listed and Create New is available. Sources: S4.
7. Click Create New > Create New API; choose OpenAPI 3.0.x and no template; enter name `sample`, version `1.0.0`, title `Sample API`, and a description; turn Auto Mock API off; then click Create API. Success signal: The definition contains `openapi: 3.0.0`, the sample info block, and `paths: {}`. Sources: S4.
8. Add server URL `https://api.example.com/v1` between the info and paths sections. Success signal: The editor contains the published `servers` block. Sources: S4.
9. Replace `paths: {}` with the tutorial's `GET /users/{userId}` operation and initial 200 response. Success signal: The generated preview updates automatically. Sources: S4.
10. Add the required integer `userId` path parameter with its description. Success signal: The preview shows the new parameter information. Sources: S4.
11. Define the 200 JSON object response with integer `id` and string `name`, then add the tutorial's 400 and 404 error descriptions. Success signal: The complete specification and generated preview include the response schema and error cases. Sources: S4.
12. Open the drop-down next to the API version and click Publish. Success signal: Swagger Studio confirms that the first API is published and its URL can be copied and shared. Sources: S4.
