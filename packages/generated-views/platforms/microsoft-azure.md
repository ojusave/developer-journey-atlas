## Documented path

1. Open Introduction to Azure for developers and select Get started with Azure dev tools under Quickstart. Success signal: The quickstart opens. Sources: S1, S2.
2. Choose VS Code for the Web or a local tool installation. Success signal: The required tool environment is available. Sources: S2.
3. For the browser route, open vscode.dev/azure and sign in with the Azure account. Success signal: The browser IDE opens under the signed-in account. Sources: S2.
4. Install the Azure Tools extension pack if all Azure tools are not already present. Success signal: Azure Tools is installed and accessible. Sources: S2.
5. Open a terminal, create a new directory, and change into it. Success signal: The terminal is in my-todo-app. Sources: S2.
6. Initialize the todo-nodejs-mongo starter template with Azure Developer CLI. Success signal: The project is initialized with the selected environment name. Sources: S2.
7. Run azd auth login and complete Azure sign-in. Success signal: The CLI is signed in. Sources: S2.
8. Run azd up, then select an Azure subscription and region when prompted. Success signal: Azure Developer CLI finishes and displays the deployed application URL. Sources: S2.
9. Open the displayed URL in a browser and verify the application is running. Success signal: The developer can see the app and add or complete tasks. Sources: S2.
10. Run azd auth status and sign in through VS Code if account details are not returned. Success signal: azd auth status returns account details or VS Code sign-in completes. Sources: S2.
11. Open the Azure view, group resources by Resource Group, expand the subscription, and find the resource group created by azd. Success signal: The created resource group is visible under the subscription. Sources: S2.
12. Expand the resource group and inspect the App Service and Cosmos DB resources. Success signal: The App Service, Cosmos DB account, and other created resources are shown. Sources: S2.
13. Start streaming logs for App Service, add a task in the to-do app, and observe the resulting log entries. Success signal: Adding a task produces entries in the streaming logs. Sources: S2.
14. Open Copilot Chat and ask how to add authentication to the Azure App Service. Success signal: Copilot returns step-by-step guidance tailored to the application. Sources: S2.
