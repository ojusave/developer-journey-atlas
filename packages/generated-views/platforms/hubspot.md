## Documented friction gates

- A standard HubSpot account is required before the Quickstart begins. Stage 2. Evidence state: documented requirement. Sources: S1, S3.
- Node.js 20+ and npm are needed for the HubSpot CLI. Stage 3. Evidence state: documented requirement. Sources: S2.
- The HubSpot CLI must be installed, potentially requiring global npm write permission. Stage 4. Evidence state: documented requirement. Sources: S1, S2.
- The Personal Access Key's scopes are bounded by the HubSpot user's permissions; Developer tools access is needed for project upload. Stage 6. Evidence state: documented requirement. Sources: S2.
- A per-user, per-account Personal Access Key authenticates the CLI and must be protected. Stage 6. Evidence state: documented requirement. Sources: S1, S2.
- HubSpot must build and deploy the project before installation. Stage 13. Evidence state: documented requirement. Sources: S1.
- The installation page displays an expected unverified-app warning and requires explicit confirmation. Stage 15. Evidence state: documented requirement. Sources: S1.
- The developer must choose a connected, test, or sandbox account for the local session. Stage 26. Evidence state: documented requirement. Sources: S1, S3.
- Missing dependencies may be installed and the local development server must start. Stage 27. Evidence state: documented requirement. Sources: S1.
- The final preview requires a contact record in the selected account. Stage 29. Evidence state: documented requirement. Sources: S1.
