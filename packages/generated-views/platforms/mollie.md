## Documented path

1. Open Set up Mollie and select the custom Payments API route. Success signal: The Accepting payments guide is linked. Sources: S1.
2. Create the Mollie account as one opaque signup step. Success signal: The Mollie dashboard opens. Sources: S1, S2.
3. Create or select a website profile and open Developers > API access tokens. Success signal: A Test API key is available. Sources: S2, S3.
4. Copy the Test API key and store it as a bearer credential. Success signal: The Test API key is copied. Sources: S3.
5. Provide a redirect URL and a webhook endpoint that returns 200 OK. Success signal: Both URLs are reachable by Mollie. Sources: S4.
6. Create a test payment with amount, description, redirectUrl, and webhookUrl. Success signal: The create-payment response succeeds. Sources: S4, S5.
7. Redirect with HTTP GET, preferably 303 See Other, to `_links.checkout` and complete the test payment. Success signal: Checkout reports payment completion. Sources: S4.
8. Send the payment-status webhook to the configured endpoint. Success signal: The endpoint returns 200 OK. Sources: S4.
9. Retrieve the payment by ID and inspect its status. Success signal: The API reports status `paid`. Sources: S4.
