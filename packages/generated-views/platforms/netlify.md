## Documented friction gates

- Netlify presents several peer starting routes and does not select a universal default. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- The correct folder depends on whether the developer is logged in. Anonymous drops publish files as-is, while logged-in drops can detect and build a framework project. Stage 3. Evidence state: documented requirement. Sources: S2.
- The managed route requires login, but the linked signup flow is not atomically documented on the public Start pages. Stage 4. Evidence state: documented requirement. Sources: S2, S3.
- The docs say deploys under 50 MB work best and that individual files over 10 MB can cause a stuck deploy. Stage 6. Evidence state: documented requirement. Sources: S2.
- Network reliability, browser version, and local free memory can block upload. The troubleshooting baseline suggests roughly four times the uncompressed project size in free memory. Stage 6. Evidence state: documented requirement. Sources: S2.
- Netlify may need to detect, build, and publish the project before the Preview URL is available. No typical duration is documented for this route. Stage 7. Evidence state: documented requirement. Sources: S2, S8.
- A production deployment consumes 15 credits. The Free plan includes 300 monthly credits with a hard limit. Stage 7. Evidence state: documented requirement. Sources: S6.
