**Developer goal:** Send a first API request in Postman, save it, and validate the response with a basic test.

**Meaningful success:** The Postman desktop app sends a GET request to Postman Echo, displays the server response, saves the request in a new collection, runs a `Status code is 200` post-response test, and shows the test result.

**Evidence boundary:** demonstrated-terminal-state. It is the final observable action of the official general Quick start and validates both the first request and a basic test before later product workflows.

## Documented path

1. Download, install, and open the Postman desktop app. Success signal: The workbench is available and can open a request tab. Sources: S1.
2. Select Add in the workbench to open a new request tab. Success signal: A new request tab is visible. Sources: S1.
3. Enter `postman-echo.com/get` as the request URL and select Send. Success signal: The lower pane displays the response returned by Postman Echo. Sources: S1.
4. Sign in to Postman before saving the request. Success signal: The developer returns to the desktop app signed in. Sources: S1, S3.
5. Select Save, create and name a new collection, then save the request into it. Success signal: The collection and request appear under Collections in the sidebar. Sources: S1.
6. Open Scripts, choose Post-response, open Snippets, and select `Status code: Code is 200`. Success signal: The script editor contains the generated status-code assertion. Sources: S1.
7. Select Send again, then open Test Results in the response section. Success signal: Test Results reports the outcome of `Status code is 200` for the returned response. Sources: S1.

## Documented friction gates

- The official Quick start requires the desktop application rather than beginning in the web app. Stage 1. Evidence state: documented requirement. Sources: S1.
- The local app requires network access to Postman Echo. Stage 3. Evidence state: documented requirement. Sources: S1.
- Saving to a collection requires a signed-in account even though the first request can be sent unsigned. Stage 4. Evidence state: documented requirement. Sources: S1, S3.

- The final request must return HTTP 200 for the generated test to pass. Stage 7. Evidence state: documented requirement. Sources: S1.
