**Developer goal:** Send a notification using Courier's API to verify integration and delivery.

**Meaningful success:** A notification message is successfully sent using the API key and appears as delivered in the Courier dashboard logs.

**Evidence boundary:** explicitly-named. The documentation explicitly defines sending a message and seeing it delivered in the dashboard logs as the first success milestone for developers integrating Courier.

## Documented path

1. Sign up or log in to Courier dashboard. Success signal: Access to Courier dashboard. Sources: S1, S5.
2. Copy API key from Settings > API Keys in the dashboard. Success signal: API key copied successfully. Sources: S1, S4, S5.
3. Set environment variable COURIER_API_KEY with the copied API key. Success signal: Environment variable accessible to SDK or CLI. Sources: S1, S5.
4. Send a notification message using curl, CLI, or SDK with the API key. Success signal: Response includes a requestId indicating message accepted. Sources: S1.
5. Open Message Logs in Courier dashboard to verify message delivery status. Success signal: Message shows delivered status in logs. Sources: S1.
