## Documented path

1. Open Rootly Quick Start and choose the dashboard test-incident route. Success signal: The Quick Start identifies the dashboard action. Sources: S2.
2. Sign up at the official Rootly signup URL. Success signal: The Rootly dashboard opens. Sources: S3, S4.
3. Open Dashboard and select Create Test Incident, or open Create Incident and enable Test Incident when available. Success signal: The incident form is visible. Sources: S2, S8.
4. Complete the required incident fields. Success signal: No required-field errors remain. Sources: S8.
5. Select Create Incident. Success signal: The newly created incident page opens. Sources: S8.
6. Verify the incident record and initial timeline entry on the incident page. Success signal: The incident page shows the record and creation state. Sources: S8.

## Documented friction gates

- Google, Slack, SSO, email and password, and invitation routes change identity and organization-entry actions. Stage 2. Evidence state: documented requirement. Sources: S3, S4.
- The signup guide does not establish that a fresh self-service account has the Rootly Admin permission required by the Slack installation guide. Stage 2. Evidence state: documented requirement. Sources: S3, S5.
- Slack setup requires Rootly Admin or Owner and Slack Workspace Admin or Owner permissions; Enterprise Grid requires an Org Owner or Admin. Stage 2. Evidence state: documented requirement. Sources: S5.
- Standard Slack, Enterprise Grid, Microsoft Teams, Google Chat, dashboard, web, and API routes have different setup and incident-creation actions. Stage 2. Evidence state: documented requirement. Sources: S1, S2, S5.
- Slack channel-creation restrictions can require additional admin scopes before Rootly can create incident channels. Stage 2. Evidence state: documented requirement. Sources: S5.
- The incident modal can enforce customized required, conditional, and lifecycle fields before Create is allowed. Stage 2. Evidence state: documented requirement. Sources: S7.
