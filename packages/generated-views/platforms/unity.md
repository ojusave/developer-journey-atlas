**Developer goal:** Install the general Unity creation environment and create the first local Unity project from the first-time-user learning template.

**Meaningful success:** Create a Unity ID with a default organization and eligible subscription, install and sign in to Unity Hub, activate a license, install Unity Editor 6.1 or later, select the Get Started with Unity learning template, name and locate a project, and create it so it appears in the Projects list and opens in the Unity Editor.

**Evidence boundary:** explicitly-named. A local project opened in the Editor is the terminal state of Unity's named first-project procedure. Completing the five guided learning sections creates a playable game, but the manual places learning after project creation under Continue learning.

## Documented friction gates

- The machine must satisfy Unity Editor operating-system, CPU, GPU, driver, memory, and non-emulated workstation or laptop requirements. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- Unity ID creation or an existing direct, third-party, or company-SSO identity is required for the Hub route. Stage 3. Evidence state: documented requirement. Sources: S1, S3, S9.
- The direct account route requires email, password, Unity Cloud username, and full name; company SSO and third-party sign-in alter these fields. Stage 5. Evidence state: documented requirement. Sources: S9.
- Account creation requires terms-of-service and privacy-policy choices and exposes an optional promotional-updates choice. Stage 6. Evidence state: documented requirement. Sources: S9.
- Unity plan eligibility depends on use case, revenue, funding, client finances for service providers, and the published Personal, Pro, Enterprise, and Industry thresholds. Stage 9. Evidence state: documented requirement. Sources: S10, S11.
- Eligible Unity Personal use is free and requires no credit card; Pro, Enterprise, and Industry routes add paid subscription or sales steps. Stage 9. Evidence state: documented requirement. Sources: S10, S11.
- Software-install, filesystem, firewall, and sometimes administrator permissions are required. Stage 12. Evidence state: documented requirement. Sources: S3, S7.
- An active Personal, named-user, serial, education, or floating Unity license must be associated with the account and machine. Stage 14. Evidence state: documented requirement. Sources: S6, S7.
- Named-user plans require an organization administrator to assign a seat before Hub sign-in can activate the license. Stage 14. Evidence state: documented requirement. Sources: S6.
- The Editor and any selected modules require a potentially large download, local storage, and install completion before a project can be created. Stage 18. Evidence state: documented requirement. Sources: S5.
- The generic first-project manual requires a template but does not mandate one; template availability and content depend on the selected Editor version. Stage 21. Evidence state: documented requirement. Sources: S1, S7.

- A writable local location is required; Unity warns that cloud-synchronized project storage is unsupported and may corrupt a project. Stage 24. Evidence state: documented requirement. Sources: S1, S7.

## Unknowns

- Which template defines the general Unity first mile? Why unresolved: The current Create your first project manual requires selecting a template and calls out Get Started with Unity as an educational option, but it does not explicitly require that template or a Core alternative. Next evidence needed: Template selection changes downloads, packages, default scenes, target platforms, and the first observable Editor content.
- Should Unity's first success be an opened project or the playable game after all five in-Editor tutorial sections? Why unresolved: The Manual explicitly names Create your first project and separates Continue learning, while Unity Learn describes its longer exercise as completing the first Unity project and making a simple playable game. Next evidence needed: The latter adds the entire 45-minute guided Editor experience and many unexposed in-product actions.
- Does the live direct-account flow add email confirmation, anti-abuse verification, or another post-submit screen before the Unity ID is usable? Why unresolved: The current official account guide documents the complete form, policy choices, Create Unity ID action, default organization, and Personal subscription, but does not describe a post-submit verification screen. Next evidence needed: Any additional live verification would add an access or wait step before Hub sign-in.
- Which operating system, eligible organizational license route, Editor release, and modules apply to the developer? Why unresolved: The official general path branches across Windows, macOS, Linux, Personal, named-user, serial, education, floating, current, archived, and module-specific routes; the roster supplies no developer or organization state. Next evidence needed: These choices produce different installer actions, administrator involvement, downloads, permissions, SDKs, and waits.
