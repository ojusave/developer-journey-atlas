**Developer goal:** Start without a Better Stack account, create the first HTTP status monitor for a URL, retain the basic single-user escalation behavior where applicable, and verify that the monitor is running and continuously checking the URL.

**Meaningful success:** A Better Stack Uptime monitor is running and continuously checking the supplied URL or IP address for an HTTP 2XX success response.

**Evidence boundary:** explicitly-named. The first running monitor is the earliest explicit functioning Uptime result; bulk imports and advanced monitors are next steps.

## Documented path

1. Open Better Stack's `Creating your first monitor` guide. Success signal: The HTTP status monitor instructions open. Sources: S1, S3.
2. Sign up for Better Stack at the official signup URL and enter Uptime. Success signal: The Uptime Monitors page can be opened. Sources: S1, S2.
3. Go to Monitors, select Create monitor, and enter a reachable URL or IP address. Success signal: The target appears in the URL to monitor field. Sources: S1.
4. Choose an alerting option. Success signal: An alerting option is selected. Sources: S1.
5. For a single-user team, leave the default escalation option selected. Success signal: The default remains selected. Sources: S1.
6. Click Create monitor. Success signal: The monitor is up and running and continuously checking the URL. Sources: S1.
