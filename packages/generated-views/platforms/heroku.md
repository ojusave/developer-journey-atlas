2. Verify the Heroku account before creating the app or using dynos by opening Account Settings, Billing, and selecting Add Credit Card. Success signal: The account is verified and app creation, deployment, and dyno actions are unlocked. Sources: S4.
3. Follow the selected tutorial through local tool installation, CLI authentication, source preparation, app creation, remote build and release, and browser verification. Success signal: The selected tutorial's build and release complete and the application loads in its documented verification interface. Sources: S2, S3.
4. Create or sign in to the Heroku account required by the Node.js guide. Success signal: The Dashboard and Account Settings open. Sources: S2.
5. Open Account Settings, Billing, select Add Credit Card, and complete account verification. Success signal: App and dyno actions are unlocked. Sources: S4.
6. Install Git, a supported Node.js runtime with npm, and the operating-system-specific Heroku CLI. Success signal: Git, Node.js, npm, and Heroku CLI run in the terminal. Sources: S2.
7. Run `heroku login`, complete browser authentication, and return to the terminal. Success signal: The CLI confirms the account email. Sources: S2.
8. Clone `https://github.com/heroku/nodejs-getting-started.git` and enter the repository directory. Success signal: The sample files are present in the current directory. Sources: S2.
9. Run `heroku create --stack heroku-26`. Success signal: The command prints the created URL and remote. Sources: S2, S4.
10. Run `git push heroku main` and wait for the remote build and release. Success signal: Output reports `Build succeeded!`, `Released`, the URL, and `Verifying deploy... done`. Sources: S2.
11. Run `heroku open` or visit the generated URL. Success signal: The official sample site loads at its `herokuapp.com` URL. Sources: S2.
