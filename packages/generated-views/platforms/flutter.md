**Developer goal:** Install Flutter with VS Code on Windows and test the developer experience by creating, running, and hot-reloading the sample counter app.

**Meaningful success:** The generated Flutter counter app runs in Chrome, retains its current counter state across a source edit, and uses the edited decrement behavior after stateful hot reload.

**Evidence boundary:** demonstrated-terminal-state. This is the last observable behavior in the page's stated quick-start achievements. The following Flutter-sidebar exploration is product orientation after the app already runs and hot reload has succeeded.

## Documented path

1. Open the Flutter quick start and keep Windows selected as the development platform. Success signal: Windows is shown as the configured platform. Sources: S1.
2. Download and install the latest Git for Windows. Success signal: Git is available for Flutter SDK cloning. Sources: S1.
3. Download and install Visual Studio Code. Success signal: VS Code launches. Sources: S1.
4. Open VS Code and install the Flutter extension from its Marketplace page. Success signal: The command palette can surface Flutter commands. Sources: S1.
5. Open View > Command Palette, type flutter, and select Flutter: New Project. Success signal: Download SDK is offered. Sources: S1.
6. Select Download SDK, choose an installation folder, and click Clone Flutter. Success signal: The SDK download completes. Sources: S1.
7. Click Add SDK to PATH after the SDK clone completes. Success signal: The notification says the Flutter SDK was added to PATH. Sources: S1.
8. Respond to the optional Google Analytics notice, then close and reopen all terminals and restart VS Code. Success signal: A new terminal can resolve Flutter tooling. Sources: S1.
9. Open the command palette again and select Flutter: New Project. Success signal: Which Flutter template? appears. Sources: S1.
10. Choose the Application template. Success signal: VS Code asks for the project parent directory. Sources: S1.
11. Select or create the parent directory and confirm Select a folder to create the project in. Success signal: VS Code prompts for an app name. Sources: S1.
12. Enter trying_flutter or another lowercase_with_underscores project name and press Enter. Success signal: VS Code starts project initialization. Sources: S1.
13. Wait while VS Code runs Flutter project initialization and retrieves dependencies. Success signal: Initialization finishes and lib/main.dart is available. Sources: S1.
14. Open lib/main.dart. Success signal: main.dart is open in the editor. Sources: S1.
15. Open the command palette, select Flutter: Select Device, and choose Chrome. Success signal: Chrome is the active device. Sources: S1, S3.
16. Select Run > Start Debugging or press F5. Success signal: Chrome opens and displays the newly created counter app. Sources: S1.
17. Click the increment button several times while the app is running. Success signal: The displayed counter increases. Sources: S1.
18. In lib/main.dart, change _counter++ to _counter--, then save all files or click Hot Reload. Success signal: The existing counter value stays the same immediately after hot reload. Sources: S1.
19. Click the counter button again and observe the value. Success signal: The retained counter value now decreases instead of increasing. Sources: S1.
