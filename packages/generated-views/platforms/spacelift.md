## Documented path

1. Open the Spacelift Quick Start and review its source-code, cloud-account, first-stack, teammate, and first-run sequence. Success signal: The page identifies Trigger your first run after four key setup steps. Sources: S1.
2. Open Start for free and choose the Europe or United States account region. Success signal: One region is selected and the four provider choices are available. Sources: S2.
3. Complete signup through one official identity provider. Success signal: The Spacelift dashboard is available. Sources: S2.
4. Integrate source code, using the official Terraform starter repository for the shortest route. Success signal: The source integration is connected. Sources: S3.
5. Configure the cloud integration required by the selected starter stack. Success signal: The cloud integration is available to stacks. Sources: S4.
6. Create a stack, connect it to the starter repository, select Terraform, and complete the summary. Success signal: The stack opens in the Stacks tab. Sources: S5.
7. Open the stack, click Trigger, then click Confirm. Success signal: The run progresses in the run view. Sources: S1.
8. Wait for the run to finish and inspect its output. Success signal: The run reaches a finished state. Sources: S1.

## Documented friction gates

- The account region must be selected before signup and later login must use the same region. Stage 2. Evidence state: documented requirement. Sources: S2, S7.
- All public signup options hand control to an external identity provider whose completion and return are not documented by Spacelift. Stage 3. Evidence state: documented requirement. Sources: S2.
- The page ties signup to agreement with the Terms of Service and Privacy Policy. Stage 3. Evidence state: documented requirement. Sources: S2.
