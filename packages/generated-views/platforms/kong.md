## Documented path

1. Open Kong Developer documentation and choose a product surface. Success signal: The developer names the specific Kong product and expected first result. Sources: S1.
2. For the representative candidate, open Get started with Kong Gateway and choose Konnect hybrid or self-managed local deployment. Success signal: Konnect or on-prem is selected. Sources: S2.
3. Start the Kong Konnect free trial. Success signal: Built-in and social identity registration options are available. Sources: S5.
4. Register and sign in with built-in username and password authentication or a Google, GitHub, or Microsoft social identity. Success signal: The Konnect UI opens for the new organization. Sources: S6.
5. Complete or defer multi-factor authentication according to the Konnect organization tier and login method. Success signal: Required MFA enrollment is complete, or the free-trial user has made the optional choice and can continue. Sources: S6.
6. On the Konnect branch, open the Konnect PAT page and select Generate Token. Success signal: A new token is available to copy. Sources: S2.
7. Export the Konnect token as KONNECT_TOKEN. Success signal: KONNECT_TOKEN is set for the quickstart command. Sources: S2.
8. Run the Kong quickstart script with the token and deck output option. Success signal: The script prints DECK_KONNECT_TOKEN, DECK_KONNECT_CONTROL_PLANE_NAME, KONNECT_CONTROL_PLANE_URL, and KONNECT_PROXY_URL exports. Sources: S2, S4.
9. Copy and paste the quickstart's printed environment-variable exports into the terminal. Success signal: decK and curl can address the provisioned environment. Sources: S2.
10. Install or update decK to version 1.64.0 or newer and confirm its version. Success signal: deck version reports 1.64.0 or newer. Sources: S2.
11. Run deck gateway ping. Success signal: The output says Successfully connected to Kong and, for Konnect, Successfully Konnected to the Kong organization. Sources: S2.
12. Create the example Gateway Service with decK. Success signal: decK applies the Service configuration. Sources: S2.
13. Create the example Route with decK. Success signal: decK applies the Route configuration. Sources: S2.
14. Proxy a request through Kong Gateway to the upstream /anything resource. Success signal: curl completes without --fail-with-body reporting an HTTP error. Sources: S2.
15. Verify the proxied response status. Success signal: HTTP 200 is returned through Kong Gateway. Sources: S2.

## Documented friction gates

- Kong documentation presents many peer products and applications without one company-wide first mile. Stage 1. Evidence state: documented requirement. Sources: S1.
- The Gateway tutorial requires choosing Konnect hybrid or self-managed local deployment. Stage 2. Evidence state: documented requirement. Sources: S2.
- Konnect requires an account before the developer can generate a personal access token; the self-service trial is $0 for 30 days with no credit card required. Stage 3. Evidence state: documented requirement. Sources: S2, S3, S5.
- Konnect registration and sign-in can use built-in username and password authentication or Google, GitHub, or Microsoft social identity. Stage 4. Evidence state: documented requirement. Sources: S6.
- MFA is mandatory for Plus and Enterprise built-in authentication, optional for a new basic-authentication free-trial signup, and recommended for all organizations. Stage 5. Evidence state: documented requirement. Sources: S6.
- The Konnect branch requires a personal access token; the self-managed Enterprise branch requires a license. Stage 6. Evidence state: documented requirement. Sources: S2, S4.
- The quickstart runs local Docker containers and depends on Docker being available and running. Stage 8. Evidence state: documented requirement. Sources: S2, S4.
- The tutorial requires decK version 1.64.0 or newer. Stage 10. Evidence state: documented requirement. Sources: S2.
- The path requires network access to the quickstart, container images, Kong or Konnect endpoints, and the httpbin upstream. Stage 12. Evidence state: documented requirement. Sources: S2, S4.
