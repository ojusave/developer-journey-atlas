**Developer goal:** Create a Vultr account, provision a first general-purpose Linux Cloud Compute instance, and verify interactive shell access.

**Meaningful success:** Vultr has provisioned the developer's first Cloud Compute instance, and the developer can open its Linux shell and verify the logged-in user.

The account guide says the user can deploy a first server after billing setup, the provisioning guide ends with Deploy, and the OpenSSH guide ends its default-credentials procedure by running `whoami`. Installing an application or configuring a production workload occurs after this first usable compute success.

## Documented friction gates

- Email verification is mandatory for the direct registration path before billing setup opens. Stage 3. Evidence state: documented requirement. Sources: S1.
- A payment method is required before the account can deploy services. The card route can use a $0.00 deposit but can trigger a temporary authorization hold; other methods require a small deposit. Stage 4. Evidence state: documented requirement. Sources: S1.
- Promotional credit is conditional. Coupon eligibility and value cannot be treated as universal account behavior. Stage 5. Evidence state: documented requirement. Sources: S1, S2.
- The first compute order requires location, plan-category, and plan decisions before the user has workload measurements. Stage 9. Evidence state: documented requirement. Sources: S3.
- Multiple image sources and operating-system versions add choice, although the standard Operating System branch is sufficient for first shell access. Stage 10. Evidence state: documented requirement. Sources: S3.
- SSH key, startup script, and firewall group are optional deployment choices. The docs warn that adding an SSH key in Console after deployment wipes the instance and its data to install the key. Stage 11. Evidence state: documented requirement. Sources: S3, S4.
- The built-in Vultr Console provides documented browser terminal access without SSH. Stage 16. Evidence state: documented requirement. Sources: S5.
- Stopping an instance does not stop normal charges. The FAQ says charges stop when the instance is destroyed. Stage 17. Evidence state: documented requirement. Sources: S6.
