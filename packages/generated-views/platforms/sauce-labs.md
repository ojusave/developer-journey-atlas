**Developer goal:** Create a Sauce Labs trial account and, if Selenium Java is selected, run the linked example test through its first visible Sauce Labs result.

**Meaningful success:** DesktopTests runs on Sauce Labs and its completed job and assets appear in Automated Test Results.

**Evidence boundary:** demonstrated-terminal-state. This is the documented remote-side completion signal of the committed Selenium Java candidate under workshop policy.

## Documented path

1. Open the free-trial signup page and choose Google, GitHub, or email signup. Success signal: The registration flow opens. Sources: S7.
2. For email signup, enter a business email and username, create a compliant password, choose United States or Europe as the test data-center location, accept the notices, and submit Sign up. Success signal: The account is created and the developer can enter Sauce Labs. Sources: S6, S7.
3. Choose the Selenium automated web-testing route and its linked Java example repository. Success signal: The repository setup and run instructions are visible. Sources: S1, S2, S4, S8.
4. Clone the example repository with git clone https://github.com/saucelabs-training/demo-java.git and import it into the IDE. Success signal: The repository modules are visible and Maven can address them. Sources: S4, S8.
5. Open User Settings and copy the Sauce Labs username and access key. Success signal: Both credential values are available for local configuration. Sources: S5, S6, S8.
6. Set SAUCE_USERNAME and SAUCE_ACCESS_KEY in the local shell, then echo both variables to verify they are present. Success signal: The verification commands print non-empty values. Sources: S5, S8.
7. Enter the best-practice module, run mvn dependency:resolve, then run mvn test-compile. Success signal: Both Maven commands complete successfully. Sources: S8.
8. Run mvn test -Dtest=DesktopTests to start the documented remote Selenium example. Success signal: Maven reports the example test result and the Remote WebDriver session ends. Sources: S4, S6, S8.
9. Open Automated > Test Results and inspect the completed Selenium job and its video or screenshot assets. Success signal: The completed job appears in Automated Test Results with its result and assets. Sources: S4, S7.

## Documented friction gates

- Email signup requires business email, username, a password meeting five composition rules, a United States or Europe data-center choice, and acceptance of the linked notices. Stage 2. Evidence state: documented requirement. Sources: S6, S7.
- Test mode, application type, framework, language, target, and data center must be selected. Stage 3. Evidence state: documented requirement. Sources: S1, S2, S3, S6.
- Automated routes require a Sauce Labs username and access key, with environment variables recommended. Stage 5. Evidence state: documented requirement. Sources: S4, S5.
- Framework-specific local dependencies and a data-center-specific endpoint must match the chosen route. Stage 4. Evidence state: documented requirement. Sources: S2, S4, S6, S8.
- Private or local applications can require a Sauce Trusted Connection route before the cloud browser or device can reach them. Stage 8. Evidence state: documented requirement. Sources: S3.
- A remote browser or device session must become available and the selected test must finish before a result can be inspected. Stage 8. Evidence state: documented requirement. Sources: S4, S6.
- The Selenium candidate requires both the test assertion result and the completed job in Automated Test Results. Stage 9. Evidence state: documented requirement. Sources: S4.
