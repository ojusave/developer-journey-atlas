## Documented path

1. Open Temporal Docs and follow Quickstart to the local language index. Success signal: Quickstarts says to choose a language to get started locally. Sources: S1, S2.
2. Run `go version` and confirm Go is installed. Success signal: The command returns a Go version. Sources: S3.
3. Run `mkdir goproject`, `cd goproject`, `go mod init my-org/greeting`, `go get go.temporal.io/sdk`, `go get go.temporal.io/sdk/client`, and `go mod tidy`. Success signal: The module resolves and tidies its dependencies. Sources: S3.
4. Run `brew install temporal` and ensure `temporal` is on PATH. Success signal: The `temporal` command is available in a new terminal. Sources: S3.
5. Open a new terminal and run `temporal server start-dev`, leaving it running. Success signal: The service listens on `localhost:7233` and the Web UI on `http://localhost:8233`. Sources: S3.
6. Create `activity.go` in package `greeting` with the published `Greet` Activity that returns `Hello <name>`. Success signal: The file defines `Greet(ctx, name)` and returns the formatted greeting. Sources: S3.
7. Create `workflow.go` with `SayHelloWorkflow`, a 10-second Activity StartToClose timeout, and `ExecuteActivity(...).Get(...)` returning the Activity result. Success signal: The Workflow returns the Activity result or its error. Sources: S3.
8. Create `worker/main.go` with the published client, Worker on `my-task-queue`, Workflow and Activity registrations, and `w.Run(worker.InterruptCh())`. Success signal: The Worker connects and registers both handlers. Sources: S3.
9. In another terminal, run `go run worker/main.go` and leave the Worker polling. Success signal: The Worker remains running and ready to process Workflow and Activity tasks. Sources: S3.
10. Create `start/main.go` with the published client, Workflow ID `greeting-workflow`, task queue `my-task-queue`, `ExecuteWorkflow`, result wait, and result log. Success signal: The code logs IDs after start and waits for the Workflow result. Sources: S3.
11. Run `go run start/main.go Temporal`. Success signal: The starter prints `Workflow result: Hello Temporal`, the Worker processes the Workflow and Activity, and details appear in the local Web UI. Sources: S3.
