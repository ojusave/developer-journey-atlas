**Developer goal:** Start without a UiPath account or project, sign up for free, enter Studio Web, use its recommended template-guided experience, run the resulting workflow on the provisioned serverless robot, and verify that the run succeeded.

**Meaningful success:** A project created from a UiPath template runs on the default Automation Cloud Serverless robot and reaches a succeeded run status.

**Evidence boundary:** explicitly-named. The first successful workflow execution is earlier than publishing and is directly named by the overview. Signup is one opaque step under policy; choosing an available recommended template is a documented in-product choice.

## Documented path

1. Open the Studio Web overview and choose its recommended template route. Success signal: Go to the Templates page is available. Sources: S1.
2. Sign up to try UiPath for free using the official registration URL. Success signal: https://studio.uipath.com/ opens. Sources: S1, S4.
3. On first open, choose the requested department and services so Studio Web can recommend templates. Success signal: The Templates page is available. Sources: S2.
4. Open an available recommended template, review its details, and select Use template. Success signal: The generated workflow opens in the designer. Sources: S1, S2.
5. Supply the template's documented required information and create or select any required application connections. Success signal: No required template configuration remains unresolved. Sources: S2.
6. Select Debug on cloud to run the project on the default Automation Cloud Serverless robot. Success signal: The Run output panel opens automatically. Sources: S3, S6.
7. Verify the cloud run completed successfully. Success signal: Run status is `succeeded` and executed activities show green checkmarks. Sources: S3, S6.

## Documented friction gates

- A UiPath account and Automation Cloud access are required to enter Studio Web. Stage 2. Evidence state: documented requirement. Sources: S1.
- The current cloud signup and application entry are client-rendered, so exact public form fields and transitions are not available. Stage 2. Evidence state: documented requirement. Sources: S1, S4.
- Designing and running Studio Web projects requires one of the documented Studio Web roles. Stage 2. Evidence state: documented requirement. Sources: S5.
- First entry asks for department and services, and the developer must choose among multiple recommended templates. Stage 2. Evidence state: documented requirement. Sources: S2.
- Some templates require external application connections that may need to be created before use. Stage 2. Evidence state: documented requirement. Sources: S2.
- Template-specific required information must be supplied before creation or completed later inside the project. Stage 2. Evidence state: documented requirement. Sources: S2.
- Each cloud run builds the project, initializes the robot machine, and transfers the project and dependencies before workflow execution. Stage 2. Evidence state: documented requirement. Sources: S6.
