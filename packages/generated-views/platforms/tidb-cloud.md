**Developer goal:** Install TiUP, start a local TiDB test cluster, and connect with the bundled TiUP client.

**Meaningful success:** TiUP Playground starts a local TiDB cluster and `tiup client` opens a SQL session against it.

**Evidence boundary:** demonstrated-terminal-state. This is the first demonstrated working database state in the official local quickstart. Loading data, TiFlash analysis, dashboards, and cleanup follow after connectivity is established.

## Documented path

1. Open Quick Start with TiDB Self-Managed and select the local test cluster route. Success signal: Deploy a local test cluster is selected. Sources: S9.
2. Install TiUP with the published curl command, then source the shell profile path printed by the installer. Success signal: The installer prints `Have a try: tiup playground`. Sources: S9.
3. Start the default local cluster. Success signal: The terminal prints `TiDB Playground Cluster is started, enjoy!` and a TiDB connection endpoint. Sources: S9.
4. Open a new terminal session and connect with the TiUP client. Success signal: The client connects to the local TiDB endpoint. Sources: S9.

## Documented friction gates

- The official signup application exposes no public form, while official prose does not enumerate all requested registration information. Stage 2. Evidence state: documented requirement. Sources: S2, S6, S8.
- Email signup requires eight or more characters with uppercase, lowercase, and numeric characters; a policy failure produces an on-screen prompt. Stage 2. Evidence state: documented requirement. Sources: S2.
- Registration does not complete until the developer confirms the email message; no timing or self-signup resend procedure is published. Stage 2. Evidence state: documented requirement. Sources: S2.
- The selected quickstart route requires permission for PingCAP and AWS Bedrock to use code snippets for research and service improvement. Stage 2. Evidence state: documented requirement. Sources: S1.
- AI-generated SQL is not guaranteed accurate; Accept with edits or Discard are the documented review controls. Stage 2. Evidence state: documented requirement. Sources: S1.
- At Starter free-quota exhaustion, new connections are denied and existing connections are throttled until quota increases or monthly reset. Stage 2. Evidence state: documented requirement. Sources: S4.
