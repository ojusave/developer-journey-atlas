## Documented path

1. Create or sign in to the Atlas account, then open or create the organization and project that will contain the first cluster. Success signal: The organization and project open and can host the cluster. Sources: S4.
2. Register an Atlas account with email, GitHub, or Google and complete the selected identity flow. Success signal: The Atlas console opens for the new user. Sources: S4.
3. Use the automatically created Atlas organization and project, or create the organization and project manually. Success signal: The project opens and can contain a cluster. Sources: S2, S4.
4. Install Atlas CLI and MongoDB Shell (`mongosh`), then connect Atlas CLI to the account and project. Success signal: Atlas CLI can address the selected organization and project. Sources: S2.
5. Run `atlas setup --force` to accept the quickstart defaults, or run `atlas setup` and answer the interactive prompts. Success signal: Atlas accepts the setup request and begins cluster creation. Sources: S2.
6. Capture and store the generated database user credentials and emitted connection string. Success signal: The credential values and connection string are available to the developer. Sources: S2.
7. Wait for Atlas to create the cluster, configure access, and load the quickstart sample data. Success signal: Output reports `Cluster created` and finishes loading sample data. Sources: S2.
8. Inspect the printed `mongosh` connection command and the quickstart completion message. Success signal: The CLI prints the connection command and the documented cluster-created terminal signal. Sources: S2.

- The developer must choose deployment product and interaction surface before a single MongoDB first-mile path exists. Stage 3. Evidence state: documented requirement. Sources: S1, S2, S3.
- Atlas paths require account, organization, and project context. Stage 3. Evidence state: documented requirement. Sources: S2, S3.
- Atlas creates or requires a database user and password, an IP access-list entry, and a connection string before client connection. Stage 3. Evidence state: documented requirement. Sources: S2, S3.
