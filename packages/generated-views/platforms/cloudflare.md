## Documented path

1. Open the Cloudflare Developer Platform learning path and follow its Workers path. Success signal: The Build applications with Cloudflare Workers route is identified. Sources: S1.
2. Open Workers Getting started, which asks the developer to build the first Worker. Success signal: The interface choices are visible. Sources: S2.
3. Select the onboarding interface whose path will define first success. Success signal: A human has selected one official route. Sources: S2, S3, S4, S5, S6, S7.
4. Open Cloudflare's signup page. Success signal: Email and password fields are visible. Sources: S8.
5. Enter an email address and password, then select Create Account. Success signal: Cloudflare creates the account and automatically sends a verification email. Sources: S8, S10.
6. Wait for Cloudflare's automatic verification email. Success signal: The recent Cloudflare message appears in the inbox or Spam folder. Sources: S9.
7. Open the verification link and log in to Cloudflare to verify the address. Success signal: My Profile shows the verified label. Sources: S9.
8. Install Node.js 16.17.0 or later. Success signal: The installed Node.js version is 16.17.0 or later. Sources: S3.
9. Run npm create cloudflare@latest -- my-first-worker. Success signal: C3 displays the project prompts. Sources: S3.
10. Choose Hello World example. Success signal: C3 advances to application type. Sources: S3.
11. Choose Worker only. Success signal: C3 advances to language selection. Sources: S3.
12. Choose JavaScript. Success signal: C3 advances to Git setup. Sources: S3.
13. Choose Yes for Git version control and No for immediate deployment. Success signal: C3 finishes and reports project files including wrangler.jsonc, src/index.js, package.json, lockfile, and node_modules. Sources: S3.
14. Run cd my-first-worker. Success signal: wrangler.jsonc and src/index.js are present. Sources: S3.
15. Run npx wrangler dev. Success signal: Wrangler starts the server for http://localhost:8787. Sources: S3.
16. If Wrangler opens a browser login, authenticate with the verified Cloudflare account and authorize Wrangler. Success signal: wrangler dev continues and serves the local Worker. Sources: S3.
17. Open http://localhost:8787 and verify the generated response is Hello World!. Success signal: The browser displays Hello World!. Sources: S3.
18. Edit src/index.js so the fetch handler returns Hello Worker!, then save the file. Success signal: src/index.js contains the new response text. Sources: S3.
19. Reload http://localhost:8787 and verify the response changes to Hello Worker!. Success signal: The browser displays Hello Worker!. Sources: S3.
20. Run npx wrangler deploy. Success signal: Wrangler reports a deployed workers.dev URL. Sources: S3.
21. Open the reported <YOUR_WORKER>.<YOUR_SUBDOMAIN>.workers.dev URL and verify Hello Worker!. Success signal: The public workers.dev endpoint returns Hello Worker!. Sources: S3.

## Documented friction gates

- The official Workers router requires an interface choice that changes the entire first-mile path. Stage 3. Evidence state: documented requirement. Sources: S2, S3, S4, S5, S6, S7.
- CLI and Dashboard deployments require a Cloudflare account; Playground preview does not, but deploying from Playground requires login or first-time onboarding. New account creation asks for email and password and automatically starts email verification. Stage 3. Evidence state: documented requirement. Sources: S3, S4, S7, S8, S9.
- The CLI path requires Node.js and installs Wrangler through C3. Stage 3. Evidence state: documented requirement. Sources: S3.
- The Prompting route requires choosing an AI tool and prompt, and the Templates route requires choosing a starter repository and launch method. Stage 3. Evidence state: documented requirement. Sources: S5, S6.
- The CLI guide states that first-time workers.dev 523 errors may require waiting a minute or so to resolve, then reloading the deployed URL. Stage 3. Evidence state: documented requirement. Sources: S3.
- New accounts receive an automatic verification email; missing or expired messages require Dashboard resend, and failing links can require another browser or private window. Stage 3. Evidence state: documented requirement. Sources: S8, S9.
- The CLI route requires npm, yarn, or pnpm plus exact C3 choices for example, Worker-only template, JavaScript, Git, and deferred deployment. Stage 3. Evidence state: documented requirement. Sources: S3.
- The Dashboard route waits for Cloudflare's hosted build and deployment before the workers.dev preview exists. Stage 3. Evidence state: documented requirement. Sources: S4.
- Workers Free is the default plan; Workers Paid is a separate optional purchase with a minimum $5 monthly account charge. Stage 3. Evidence state: documented requirement. Sources: S10.
