## Documented path

1. Open the LangChain Python overview and follow Get started to Install, then Quickstart. Success signal: The Quickstart title Build your first agent in minutes is visible. Sources: S1, S3, S2.
2. Initialize a uv project. Success signal: uv creates project metadata in the working directory. Sources: S2.
3. Add langchain and deepagents with uv. Success signal: uv resolves and records the dependencies. Sources: S2.
4. Run uv sync. Success signal: uv sync completes successfully. Sources: S2.
5. Obtain an API key from one supported model provider. Success signal: The required provider credential is available. Sources: S2.
6. Export the environment variable or variables required by the chosen provider. Success signal: The relevant environment variable is set in the current shell. Sources: S2.
7. Create runnable Python code that imports create_agent and defines get_weather. Success signal: The module can import LangChain and the function is defined. Sources: S2.
8. Create the agent with the selected model, get_weather tool, and helpful-assistant system prompt. Success signal: create_agent returns an agent instance. Sources: S2.
9. Invoke the agent with the documented San Francisco weather message. Success signal: The model processes the message and selects the weather tool. Sources: S2.
10. Print the final message content blocks. Success signal: The final message content_blocks are printed. Sources: S2.
11. Verify that the agent used the weather tool for San Francisco and returned its final response. Success signal: The output shows the get_weather tool interaction and final content, completing the first-agent milestone. Sources: S2.

## Documented friction gates

- Python 3.10+ and one package-management workflow are required. Stage 2. Evidence state: documented requirement. Sources: S2, S3.
- Most hosted model choices require an external provider account and key; the guide does not decompose provider onboarding. Stage 5. Evidence state: documented requirement. Sources: S2.
- Provider secrets must be placed in the correct environment variables. Stage 6. Evidence state: documented requirement. Sources: S2.
- The invocation depends on provider authentication, service availability, model access, and usage capacity outside LangChain. Stage 9. Evidence state: documented requirement. Sources: S2.
