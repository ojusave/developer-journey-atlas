**Developer goal:** Create or obtain access to an F5 Distributed Cloud SaaS account, enter the tenant Console, set initial work-domain and skill-level preferences, and reach the populated Home page.

**Meaningful success:** The developer has entered the assigned F5 Distributed Cloud tenant, accepted initial terms if required, selected work domains and skill level, and reached Console Home with workspace tiles.

**Evidence boundary:** explicitly-named. The general Console quickstart ends its initial setup when Home loads. Opening a workspace or deploying a service follows later.

## Documented path

1. Open the Distributed Cloud Quickstart index and choose the general Console onboarding guide. Success signal: Get Started with Distributed Cloud Console is available as the general route. Sources: S1.
2. Open the Console guide and review its SaaS-account prerequisite and Home-page objective. Success signal: The guide explicitly requires an existing SaaS account. Sources: S2.
3. Sign up for an F5 Distributed Cloud SaaS account. Success signal: The developer has a tenant name and can open Console. Sources: S2, S3.
4. Enter the tenant name and sign in with SaaS credentials or configured SSO. Success signal: The work-domain screen opens. Sources: S2.
5. Choose work domains and continue. Success signal: The skill-level screen opens. Sources: S2.
6. Choose Intermediate User or Advanced User and click `Next`. Success signal: Console Home loads. Sources: S2.
7. Verify the Console Home page. Success signal: Home displays Common Workspaces reflecting the selected domains and skill level. Sources: S2.

- Login requires a tenant name and SaaS credentials or configured SSO. Stage 3. Evidence state: documented requirement. Sources: S2.
- An insufficient role triggers an administrator access request and wait. Stage 3. Evidence state: documented requirement. Sources: S2.
- First login can require acceptance of terms and conditions. Stage 3. Evidence state: documented requirement. Sources: S2.
