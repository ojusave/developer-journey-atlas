**Developer goal:** Build and visibly verify a first Atlassian cloud app without a preselected Atlassian product.

**Meaningful success:** The app panel is visible on the Jira issue with the configured title and hello-world content.

**Evidence boundary:** demonstrated-terminal-state. Selected under workshop fastest/most-used policy; terminal taken from the committed official candidate route.

## Documented path

1. Complete the shared Forge setup: install Node.js LTS 22.x or 24.x and verify it, install Forge CLI and verify it, create a scoped Forge API token, authenticate, and create or finish a cloud developer site. Success signal: Forge CLI can run authenticated commands and the developer site is ready. Sources: S1.
2. Choose Bitbucket, Confluence, Jira, or Jira Service Management and enter that product's linked hello-world tutorial. Success signal: No common visible app signal exists until the product route is selected. Sources: S1, S2, S3.
3. Install Node.js LTS 22.x or 24.x and run `node --version`. Success signal: The version command reports 22.x or 24.x. Sources: S1.
4. Run `npm install -g @forge/cli`, then run `forge --version`. Success signal: The CLI prints its version. Sources: S1.
5. Create or sign in to the Atlassian account and create a scoped Forge API token with name, expiry, Forge app selection, and recommended or custom scopes. Success signal: The full token is available for CLI login. Sources: S1.
6. Run `forge login`, answer the analytics prompt, and authenticate with the account email and scoped token. Success signal: Forge accepts authenticated commands. Sources: S1.
7. Create the free cloud developer site with the account email, wait for it to be ready, sign in, and complete the setup wizard. Success signal: The site setup wizard is complete and Jira opens. Sources: S1, S4.
8. Run `forge create`, name the app, and select `development` when Forge prompts for the default environment. Success signal: Forge advances to the Atlassian app selection after accepting `development`. Sources: S4.
9. Select Jira as the Atlassian app, select UI Kit as the category, select `jira-issue-panel` as the template, and enter the generated app subdirectory. Success signal: The app directory contains the manifest, package files, and generated source. Sources: S4.
10. Edit `manifest.yml` and change the issue-panel title to `Forge app for <your name>`. Success signal: The manifest saves with the tutorial's title change. Sources: S4.
11. Run `forge deploy` from the app's top-level directory and wait for build, compilation, and deployment. Success signal: Forge reports a successful deploy without compilation errors. Sources: S4.
12. Run `forge install`, select Jira as the Atlassian context, enter the developer-site URL, and complete any prompted consent. Success signal: The CLI displays the successful installation message. Sources: S4.
13. Create a Jira issue, open it, select the Apps grid button, and select the installed app. Success signal: The app panel is visible on the Jira issue with the configured title and hello-world content. Sources: S4.

## Documented friction gates

- Supported Node.js LTS installation and `node --version` verification must precede npm-based Forge CLI installation and `forge --version` verification. Stage 1. Evidence state: documented requirement. Sources: S1.
- A scoped Forge API token with expiry and appropriate scopes must be created, copied, and used for CLI authentication. Stage 1. Evidence state: documented requirement. Sources: S1.
- Keychain and libsecret availability determines whether `forge login` or environment variables are used; root execution is prohibited. Stage 1. Evidence state: documented requirement. Sources: S1.
- A cloud developer site must be created and become ready before its setup wizard can complete. Stage 1. Evidence state: documented requirement. Sources: S1.
- Bitbucket, Confluence, Jira, or Jira Service Management must be selected for one visible first app. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
