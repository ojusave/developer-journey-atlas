## Documented path

1. Open the Dagster documentation home and follow the Quickstart link to Build your first Dagster pipeline. Success signal: The guide begins with Step 1: Scaffold a new Dagster project. Sources: S1, S3.
2. Install Python 3.10 or higher and install `uv` with the operating-system command shown in the Dagster installation guide. Success signal: The environment can invoke `uvx` for the next scaffold command. Sources: S1, S2.
3. Run `uvx create-dagster@latest project dagster-quickstart` and answer `y` when prompted to run `uv sync`. Success signal: The generated project has the structure displayed by the guide. Sources: S1, S2.
4. Change to `dagster-quickstart`, activate `.venv` with the operating-system command, and run `uv add pandas`. Success signal: The environment provides `dg` and the project can import pandas. Sources: S1, S2.
5. Run `dg scaffold defs dagster.asset assets.py`. Success signal: The `defs` directory contains `assets.py`. Sources: S1.
6. Create `src/dagster_quickstart/defs/data/sample_data.csv` and copy the guide's four-row sample dataset into it. Success signal: The file contains the documented header and four rows. Sources: S1.
7. Replace `assets.py` with the guide's `processed_data` asset definition. Success signal: The file contains the documented `processed_data` asset definition and its input and output paths. Sources: S1.
8. From the project root, run `dg dev` and open `http://localhost:3000`. Success signal: The browser shows the Dagster UI and its Assets tab. Sources: S1.
9. Open Assets, click View lineage, and click Materialize. Success signal: A run is available from the Runs tab. Sources: S1.
10. Open the Runs tab and click View for the executing run. Success signal: The run view is displayed. Sources: S1.
11. Run `cat src/dagster_quickstart/defs/data/processed_data.csv`. Success signal: The output matches the documented header and Young, Middle, Senior, and Middle values, establishing the first pipeline result. Sources: S1.
