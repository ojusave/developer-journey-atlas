## Documented path

1. Open a terminal in the target project directory and choose existing-account authentication or new-account signup. Success signal: The developer has the correct wizard command and project directory. Sources: S1, S2, S3.
2. Run `npx -y @posthog/wizard@latest` for an existing account, or the documented signup command with `--signup`, email, and `us` or `eu` region. Success signal: The wizard begins authentication and codebase analysis. Sources: S1, S3.
3. Authenticate the existing account or provision the new account and project. Success signal: The wizard has credentials for the created or selected PostHog project. Sources: S1, S3.
4. Let the wizard detect the framework and install the relevant PostHog SDKs. Success signal: Required SDK packages are present in the codebase. Sources: S1, S2.
5. Let the wizard scan the app architecture, write project host and API key to environment configuration, instrument client and server SDKs, and add custom events based on real product flows. Success signal: The application contains SDK initialization and generated event instrumentation tied to the PostHog project. Sources: S1.
6. Allow the wizard to generate PostHog insights and dashboards, wait for it to finish, and review its completion state. Success signal: The wizard exits successfully and reports completion; the documentation's project-onboarded state is reached. Sources: S1, S2.
