## Documented friction gates

- An Exa account and Dashboard access are required, but the public quickstart does not document the signup and verification sequence. Stage 2. Evidence state: documented requirement. Sources: S2, S3.
- A new account must complete onboarding before Exa grants the documented $10 free-credit balance used by the selected first request; a card or purchased credits are not required for this route. Stage 3. Evidence state: documented requirement. Sources: S6.
- A secret API key must be obtained, stored outside source control, and available to the selected runtime. Stage 4. Evidence state: documented requirement. Sources: S2, S3.
- Python and JavaScript routes require different SDK, environment-loader, and optional OpenAI packages; cURL skips this gate. Stage 6. Evidence state: documented requirement. Sources: S2.
- The quickstart requires both a language or cURL choice and a Search, Answer, or Chat Completions use-case choice. Stage 7. Evidence state: documented requirement. Sources: S2.
- Requests fail with 402 when account credits are exhausted or API-key or team budgets are exceeded; purchased credits or auto recharge become conditional recovery paths. Stage 8. Evidence state: documented requirement. Sources: S4, S6.
