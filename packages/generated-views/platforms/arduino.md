**Developer goal:** Program a connected Arduino board for the first time with the current desktop Arduino IDE.

**Meaningful success:** Arduino IDE compiles and uploads a sketch to the selected board, and the sketch starts running on the hardware.

**Evidence boundary:** demonstrated-terminal-state. This is the first end-to-end proof across IDE installation, USB connection, board support package, board and port selection, compilation, transfer, and execution. Custom projects and cloud features follow later.

## Documented path

1. Download the latest Arduino IDE 2 release for the computer's operating system and install or launch it using the official OS-specific instructions. Success signal: Arduino IDE opens successfully. Sources: S2, S3.
2. Connect the Arduino board directly to the computer with a compatible data USB cable. Success signal: The board powers on and appears as a port or unknown connected device in the IDE. Sources: S1, S4.
3. Install the board package when Arduino IDE reports that support files for the detected board are missing. Success signal: The board can be selected without a missing-platform warning. Sources: S1.
4. Select the correct board and serial port using the IDE 2 board selector. Success signal: The board selector displays the intended board and port. Sources: S1, S4, S6.
5. Open a sketch, using the guide's built-in Blink example through File > Examples > 01.Basics > Blink for a no-authoring first run. Success signal: The intended sketch is loaded and ready to compile. Sources: S1, S5.
6. Optionally click Verify to compile first, then click Upload and wait while the IDE compiles and programs the board. Success signal: The upload completes without a failed-upload error and the IDE no longer reports the transfer in progress. Sources: S1, S6.
7. Observe the uploaded sketch starting on the board. Success signal: The expected sketch behavior is visible or otherwise observable on the hardware; for Blink, the built-in LED changes state repeatedly on a compatible board. Sources: S1, S5.

## Documented friction gates

- The current desktop IDE is a separate OS-specific download and installation. Stage 1. Evidence state: documented requirement. Sources: S2, S3.
- A compatible board, data-capable cable, connector, and usable computer port are required. Stage 2. Evidence state: documented requirement. Sources: S1, S4.
- Each board family can require a separate board package and drivers before compilation and upload. Stage 3. Evidence state: documented requirement. Sources: S1, S4.
- USB enumeration and serial-port access can require OS permission or Linux configuration. Stage 4. Evidence state: documented requirement. Sources: S4, S6.
- The exact board model and serial port must match the connected hardware. Stage 4. Evidence state: documented requirement. Sources: S1.
- Board-package installation, compilation, and transfer must finish before execution can be checked. Stage 6. Evidence state: documented requirement. Sources: S1.
- A clean upload is followed by a hardware behavior check; the sketch starting on the board is the guide's terminal state. Stage 7. Evidence state: documented requirement. Sources: S1, S5.
