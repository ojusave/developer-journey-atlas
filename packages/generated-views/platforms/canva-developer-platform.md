**Developer goal:** Create and preview the Canva Apps SDK Hello World app through the official CLI route.

**Meaningful success:** The generated public Hello World Canva app runs on localhost, opens inside a Canva preview editor, and responds to its included button.

**Evidence boundary:** demonstrated-terminal-state. The next numbered section begins editing application code, so the interactive preview is the final demonstrated success before customization starts.

## Documented path

1. Create or use a free Canva account that can access the Developer Portal. Success signal: The developer can authenticate to Canva and its Developer Portal. Sources: S1, S2.
2. Install or verify Git, Node.js v24, and npm v11. Success signal: git, node, and npm commands are available at the documented versions. Sources: S1, S2.
3. Run npm install -g @canva/cli@latest. Success signal: The canva command is available. Sources: S1.
4. Run canva login. Success signal: The Canva CLI access page opens. Sources: S1.
5. Click Allow to grant the Canva CLI permission to manage Canva apps. Success signal: The browser displays a confirmation code. Sources: S1.
6. Copy the browser confirmation code and paste it into the Canva CLI prompt. Success signal: The CLI accepts the code and can manage the developer's apps. Sources: S1.
7. Run canva apps create "My New App" --template="hello_world" --distribution="public" --git --installDependencies. Success signal: The build process completes and the CLI opens the Developer Portal to the new app's configuration page. Sources: S1, S3.
8. Authenticate to the Developer Portal if the CLI-opened configuration page requests login. Success signal: The app's configuration and Code upload navigation are available. Sources: S1.
9. Change into the generated my-new-app directory. Success signal: package.json and app source are in the current directory. Sources: S1.
10. Run npm start. Success signal: The server reports that it is running at http://localhost:8080. Sources: S1.
11. In the Developer Portal, navigate to the app's Code upload page. Success signal: App source controls are visible. Sources: S1.
12. Select App source > Development URL and enter http://localhost:8080. Success signal: The Development URL field contains the running server URL. Sources: S1.
13. Click Preview to open the app in a preview Canva editor. Success signal: A preview editor opens for the app. Sources: S1.
14. Click Open if this is the first preview for the app. Success signal: The example app UI is visible. Sources: S1.
15. If Chrome prompts for local-network access, click Allow. Success signal: The local app finishes loading in the preview. Sources: S1, S2.
16. Use the button included in the Hello World example app. Success signal: The button responds with Hello world! as documented. Sources: S1, S3.

## Documented friction gates

- A Canva account and Developer Portal access are required before CLI authorization and app creation. Stage 1. Evidence state: documented requirement. Sources: S1, S2.
- The route requires a modern browser, terminal, Git, exact documented Node.js and npm major versions, then the Canva CLI and app dependencies. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- The developer must authorize the Canva CLI to manage Canva apps and transfer a browser confirmation code back to the terminal. Stage 5. Evidence state: documented requirement. Sources: S1.
- App audience is consequential: public works for the selected route, while private requires an Enterprise team and administrator review. Stage 7. Evidence state: documented requirement. Sources: S1.
- A local server must remain running on port 8080, but it is only usable through Canva's preview flow. Stage 10. Evidence state: documented requirement. Sources: S1.
- Chrome may require local-network permission before the Canva-hosted editor can connect to localhost. Stage 15. Evidence state: documented requirement. Sources: S1, S2.
