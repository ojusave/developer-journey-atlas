**Developer goal:** Start without a Fivetran account, create and verify a free-trial identity, connect a supported data source and owned destination, pass both connection tests, start the first initial sync, and receive Fivetran's sync-complete notification.

**Meaningful success:** Fivetran has connected a selected source to a selected destination and completed the connection's initial data sync.

**Evidence boundary:** explicitly-named. Initial-sync completion is the first documented proof that source authentication, destination authentication, both connection tests, optional schema selection, and data movement worked. Incremental sync evaluation, transformations, activations, usage estimates, billing, and ongoing scheduling are later lifecycle work.

## Documented path

1. Open the Quickstart Guide to Fivetran and select its Sign up for Fivetran sequence. Success signal: The guide directs the developer to `https://fivetran.com/signup`. Sources: S1, S3.
2. Sign up for Fivetran and verify the account. Success signal: The `Welcome to Fivetran` page opens. Sources: S1, S2, S5.
3. Skip or answer the trial-priority prompt, then choose Google Sheets as the source. Success signal: The embedded source setup guide appears. Sources: S1, S2, S3.
4. Authorize and configure the Google Sheets connection, then click `Save & Test`. Success signal: The source connection tests pass; click `Continue`. Sources: S1, S2.
5. Select the existing supported destination, complete its embedded setup guide, and click `Save & Test`. Success signal: The destination connection tests pass; click `Continue`. Sources: S1, S2.
6. Select source data if the connector requests it, then save the configuration. Success signal: `Start initial sync` is available. Sources: S2.
7. Click `Start initial sync`. Success signal: The connection status shows the sync in progress. Sources: S1, S2, S4.
8. Wait for and verify Fivetran's initial-sync completion notification. Success signal: Fivetran emails that the initial sync has finished and the dashboard reports the connection's synced status. Sources: S1, S2, S4.

## Documented friction gates

- The general first-mile path requires a new Fivetran account before the guided connector setup. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- Fivetran sends a confirmation email whose link must be followed before account completion. Stage 3. Evidence state: documented requirement. Sources: S1, S2.
- The live initial account form is client-rendered and its exact public fields are not enumerated by the written guides. Stage 3. Evidence state: documented requirement. Sources: S1, S2, S5.
- The developer must select a source and destination, and their choices change required fields, authorizations, and schema-selection steps. Stage 3. Evidence state: documented requirement. Sources: S1, S2.
- Both source and destination setup forms require system-specific connection details described by embedded guides. Stage 3. Evidence state: documented requirement. Sources: S1, S2.
- The initial sync runs until Fivetran reports it finished and sends its completion notification. Stage 3. Evidence state: documented requirement. Sources: S1, S2.
- Only Fivetran users with particular roles can add new connections. Stage 3. Evidence state: documented requirement. Sources: S4.
