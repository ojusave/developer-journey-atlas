**Developer goal:** Start without a Checkly account, add Monitoring as Code to an existing project with the vendor-recommended quickstart, authenticate, test the generated checks, deploy them, and verify that the checks are live in the Checkly dashboard.

**Meaningful success:** The project's generated or edited checks pass a Checkly cloud test, are deployed, and appear in the dashboard ready for scheduled monitoring.

**Evidence boundary:** demonstrated-terminal-state. Deployed checks visible in the dashboard are the first durable monitoring result; alert channels and CI integration follow.

## Documented path

1. Open the Checkly Quickstart and select `Set it up yourself`. Success signal: Bootstrap, login, check, test, and deploy sections are available. Sources: S1.
2. In an existing project, run `npx checkly init`; decline agent setup, allow npm install, and add demo checks. Success signal: The CLI prints `All done!`. Sources: S1.
3. Run `npx checkly login` and open the browser authentication URL. Success signal: The browser presents sign-in or signup. Sources: S1, S2.
4. Create a free Checkly account through the official signup flow and return to the CLI. Success signal: Login succeeds and `npx checkly whoami` identifies the account. Sources: S2, S3.
5. Edit `__checks__/homepage.spec.ts` to load `https://www.checklyhq.com`, assert status below 400, assert the title matches Checkly, and save a screenshot. Success signal: The file contains all three published assertions/actions. Sources: S1.
6. Run `npx checkly test`. Success signal: The terminal reports `2 passed, 2 total` for homepage loads and Hello API. Sources: S1.
7. Run `npx checkly deploy` and confirm deployment. Success signal: The deploy command completes. Sources: S1.
8. Open the Checkly dashboard. Success signal: The checks are ready to start monitoring. Sources: S1.
