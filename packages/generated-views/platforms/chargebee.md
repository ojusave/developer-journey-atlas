**Developer goal:** Create a Chargebee Billing test site and follow the API Quickstart until the selected SDK creates the sample subscription and associated paid invoice.

**Meaningful success:** The test-site API response contains the created active subscription, its customer, the charged test card, and the generated paid invoice.

**Evidence boundary:** explicitly-named. The Quickstart explicitly ends after creating the sample subscription and returning its customer, card, and paid invoice. Later billing features and production setup are outside this first API route.

2. Open the Chargebee trial signup page and enter the required business values and data-center choice. Success signal: All required business, monetization, employee-count, and data-center inputs are populated. Sources: S3.

6. Open the API Quickstart and its Chargebee free-trial link. Success signal: The form is visible. Sources: S1, S3.

8. Choose the optional marketing setting, acknowledge the Terms of Service and Privacy Notice, and select Complete Signup. Success signal: The Complete Account Setup email flow begins. Sources: S2, S3.
9. Send the Complete Account Setup email. Success signal: The email arrives at the registered business address. Sources: S2.
10. Open the email link, claim the account, and set the password, unique site domain, and company name. Success signal: The owner can log in to the test site. Sources: S2, S6.
11. Select Skip the setup when Chargebee offers pricing-page auto-population. Success signal: Chargebee takes the developer to the test-site dashboard. Sources: S2.
12. Go to Settings, Configure Chargebee, API Keys and Events, API keys; select Add API Key, choose Full-Access Key, name it, and select Create Key. Success signal: The new full-access key is listed for the test site. Sources: S1, S5, S6.
13. Install the Node Chargebee SDK with the published npm, yarn, or pnpm command. Success signal: The package is available to the application. Sources: S1.
14. Initialize the Chargebee client with the test-site name and full-access API key. Success signal: The client is configured for the test site. Sources: S1.
15. Run the published customer-create call for `acme-east` and `Acme Eastern`, enable auto-collection, and attach the published test card. Success signal: The response returns `customer` and `card` objects. Sources: S1.
16. Run the published item-family call to create `cloud-storage` named Cloud Storage. Success signal: The response returns the `item_family` object. Sources: S1.
17. Run the published item call to create the `silver-plan` plan item inside the Cloud Storage family. Success signal: The response returns the `item` object. Sources: S1.
18. Run the published item-price call to create `silver-plan-USD-monthly` at 50000 minor units every month. Success signal: The response returns the `item_price` object. Sources: S1.
19. Run the published create-with-items call to subscribe `acme-east` to five units of `silver-plan-USD-monthly`. Success signal: The API returns subscription, customer, card, and invoice objects. Sources: S1.
20. Verify that the returned subscription is `active`, the card was charged, and the generated invoice is `paid`. Success signal: The response matches the Quickstart's subscription, customer, card, and paid-invoice terminal. Sources: S1, S4.

- Submitting the trial acknowledges the Chargebee Terms of Service and Privacy Notice. Stage 2. Evidence state: documented requirement. Sources: S3.

- The site documentation requires claiming the account from the Complete Account Setup email. Stage 4. Evidence state: documented requirement. Sources: S2.

- Chargebee can offer Proceed or Skip for AI-populated pricing-page setup before the test-site dashboard. Stage 5. Evidence state: documented requirement. Sources: S2.
