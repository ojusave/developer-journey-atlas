**Developer goal:** Create a Perplexity account and API group, fund API usage, generate a key, and run the official Python Agent API first call.

**Meaningful success:** The Python program sends the published low-preset Agent API request and prints its web-grounded response text.

## Documented path

1. Open the API Quickstart and follow its API Portal path. Success signal: The developer can choose an account sign-in method. Sources: S1, S3.
2. Choose email signup and enter the email address. Success signal: The email authentication message is received. Sources: S2, S8.
3. Open the secure login link or enter the one-time password to authenticate. Success signal: The API Portal can be accessed. Sources: S2, S9.
4. Create the first API group and choose a descriptive group name. Success signal: Group Settings, Billing, and API keys are available. Sources: S3, S4.
5. Open Group Settings and complete the organization name, address, and tax details. Success signal: The group is configured for billing. Sources: S3.
6. Open Billing, select Add payment method, and enter the requested credit-card information in the Stripe-managed flow. Success signal: The Billing page shows the payment method and credit balance. Sources: S3, S6.
7. Select Buy more credits and purchase enough prepaid API credits for the call. Success signal: A positive credit balance is shown. Sources: S3, S5, S6.
8. Open API keys and select + Generate API Key. Success signal: A new key is visible for copying. Sources: S3, S4.
9. Copy the key immediately and store it securely before leaving the page. Success signal: PERPLEXITY_API_KEY can be set locally. Sources: S4.
10. Run `pip install perplexityai`. Success signal: The Perplexity class can be imported. Sources: S1.
11. Export the copied key as `PERPLEXITY_API_KEY`. Success signal: The next program can initialize Perplexity without an inline key. Sources: S1.
12. Run the published Python Agent API example with preset `low` and print `response.output_text`. Success signal: The response is completed and contains assistant output_text plus citations and usage data. Sources: S1, S5.

## Documented friction gates

- Email signup depends on receiving and using a login link or one-time password. Stage 2. Evidence state: documented requirement. Sources: S2.
- API group setup requires organization name, address, and tax details. Stage 5. Evidence state: documented requirement. Sources: S3.
- The official API group prerequisites require a credit card for payment setup. Stage 6. Evidence state: documented requirement. Sources: S3.
- Credits are purchased in advance, and the public docs do not state a minimum purchase amount for this first call. Stage 7. Evidence state: documented requirement. Sources: S3, S5.
- The generated key is shown only once and cannot be retrieved later. Stage 9. Evidence state: documented requirement. Sources: S4.
