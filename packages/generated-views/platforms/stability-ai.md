**Developer goal:** Create a Stability AI Developer Platform account with free credits and an API key, then run the Stable Image Core Python sample until `lighthouse.webp` is saved.

**Meaningful success:** The Stable Image Core sample returns HTTP 200 and saves the generated lighthouse image as `lighthouse.webp`.

**Evidence boundary:** demonstrated-terminal-state. The saved image is the first durable proof that account credits, API-key authentication, generation, and response handling work end to end.

## Documented path

1. Open Getting Started and select Login from the Developer Platform documentation. Success signal: The official platform.stability.ai account surface is reached. Sources: S1, S6.
2. Complete Google signup through the official Developer Platform. Success signal: The account dashboard opens and shows credits. Sources: S1, S6, S7.
3. Open API Keys and generate a key with the plus control. Success signal: The new key is copied for use. Sources: S1, S2.
4. Create a Python environment, install `requests`, and expose the API key securely. Success signal: Python imports `requests` and reads the key. Sources: S4.
5. Run the official Stable Image Core Python sample with the lighthouse prompt and WebP response. Success signal: The response status is HTTP 200. Sources: S4.
6. Verify that the sample wrote `lighthouse.webp` and open it. Success signal: `lighthouse.webp` exists and renders as an image. Sources: S4.

## Documented friction gates

- The selected Google social-login route continues through a client application whose provider consent and account-convergence sequence are not public in the official guide. Stage 2. Evidence state: documented requirement. Sources: S1, S6.
- After login, the public client defines a required checkbox agreeing to the Terms, acknowledging the Privacy Policy, and confirming age 18; Accept submits it, Decline prevents API use, and marketing opt-in is optional. Stage 2. Evidence state: documented requirement. Sources: S6, S7.
- Stability AI directs developers to store API keys in a secret manager or environment variable and never expose them in client-side code or public repositories. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- Stable Image Core charges 3 credits only for a successful generation; failed generations are not charged. Stage 2. Evidence state: documented requirement. Sources: S4.
- A `403` response means the request was flagged by content moderation. Stage 2. Evidence state: documented requirement. Sources: S4.
- The API is limited to 150 requests per 10 seconds; exceeding the limit returns `429`. Stage 2. Evidence state: documented requirement. Sources: S4.
