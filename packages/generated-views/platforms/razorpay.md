**Developer goal:** Create a Razorpay account, choose the product and integration matching the business, and complete its first simulated Test Mode transaction.

**Meaningful success:** The developer completes one simulated Razorpay transaction using the product route selected for the business.

**Evidence boundary:** explicitly-named. The general Quickstart says to choose a product and test before going live. The exact implementation and signal vary by that choice, so the success class is documented while a universal chronological route is not.

## Documented path

1. Open Razorpay's Payments Quickstart. Success signal: The guide identifies account creation as Step 1 and product selection as Step 2. Sources: S1.
2. Complete Basic Details and continue. Success signal: Business type and subtype controls are shown. Sources: S2.
3. Select registered or unregistered business, choose the business subtype, and complete the matching business-detail branch. Success signal: The account proceeds into CKYC or traditional KYC. Sources: S2.
4. Complete the applicable CKYC or traditional KYC submission path, add and verify bank details, submit the application, and choose Explore Test Dashboard. Success signal: The page offers Explore Test Dashboard while Live Mode remains pending verification. Sources: S1, S2.
5. Open Dashboard in Test Mode while KYC verification is pending or completed. Success signal: The Dashboard exposes Test Mode and product configuration. Sources: S1, S2.
6. Select the Razorpay product and integration method that matches business needs and technical capabilities. Success signal: No single official universal next step exists at this decision. Sources: S1.
7. Open Razorpay's Payments Quickstart and follow the signup link. Success signal: The contact field is available. Sources: S1, S2.
8. Enter email or a 10-digit phone number, create a password, enter the delivered OTP, click Verify, click Submit OTP, choose unregistered individual business, and click Proceed. Success signal: The full-name field appears. Sources: S2.
9. Enter full name, mobile number, and mobile OTP, selecting Continue after each action. Success signal: The accepting-payments platform choices appear. Sources: S2.
10. Select website as the platform, continue, add the website and social links or defer them, review Basic Details, and continue. Success signal: Registered and unregistered choices appear. Sources: S2.
11. Select unregistered and the individual subtype, enter personal PAN, confirm the fetched PAN details, select or enter the brand name, review the business details, and continue. Success signal: Razorpay records the individual-business details and requests CKYC or traditional KYC information. Sources: S2.
12. Complete the available unregistered-individual KYC submission path, add bank details, submit the application, and click Explore Test Dashboard. Success signal: Explore Test Dashboard is available before Live Mode activation. Sources: S1, S2.
13. In Test Mode, open Account & Settings, open API Keys under Website and app settings, and click Generate Key. Success signal: The credential download is available. Sources: S1, S3.
14. Download and securely save the Key ID and Key Secret. Success signal: Both credential values are retained. Sources: S1, S3.
15. Create an order on the server with a Test Mode authenticated POST to `/v1/orders`. Success signal: The server retains the returned order id for Checkout. Sources: S4.
16. Add a Pay button and load `https://checkout.razorpay.com/v1/checkout.js` on the web page. Success signal: The page displays the Pay button. Sources: S4.
17. Create Razorpay Checkout options with the Test Key ID, amount, currency, business name, description, returned order_id, handler or callback, optional prefill, notes, and theme. Success signal: The button click handler can call `open()`. Sources: S4.
18. Click the Pay button and complete a simulated payment using Razorpay's Test Mode payment data. Success signal: The success handler or callback runs. Sources: S4.
19. Return razorpay_payment_id, razorpay_order_id, and razorpay_signature to the configured success handler or callback. Success signal: All three fields are present. Sources: S4.
20. Verify the payment signature on the server using the order id, payment id, and Key Secret. Success signal: The generated and returned signatures match. Sources: S4.
21. Open Dashboard, navigate to Transactions and Payments, and locate the generated payment id. Success signal: The payment status is `Captured`. Sources: S4.

## Documented friction gates

- Razorpay requires an account with verified contact information. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- Signup requires OTP verification for the initial contact and mobile number. Stage 2. Evidence state: documented requirement. Sources: S2.
- Business type and subtype determine PAN, document, CKYC, and KYC actions. Stage 4. Evidence state: documented requirement. Sources: S2.
- The documented India setup path requires KYC submission and bank verification before it offers Explore Test Dashboard; Test Mode integration can then proceed while KYC verification is in progress. Stage 5. Evidence state: documented requirement. Sources: S1, S2.
- The general Quickstart requires the developer to choose among materially different products and integration methods. Stage 7. Evidence state: documented requirement. Sources: S1.
