## Documented path

1. Open the Vonage platform getting-started overview and review the common account, authentication, tool, webhook, and phone-number concepts. Success signal: No API result is produced at platform level. Sources: S1.
2. Select the target API, operation, authentication mode, and curl, Postman, or SDK surface. Success signal: No common terminal signal is documented. Sources: S1, S2, S3.
3. Sign up for a Vonage API account and open the Developer Dashboard. Success signal: The Dashboard shows the account credentials and balance. Sources: S1.
4. Open Dashboard Settings and set SMS settings to `Messages API`. Success signal: The SMS setting is saved as Messages API. Sources: S2.
5. Create a Vonage Application for Messages API credentials and configuration. Success signal: The application appears in the Developer Dashboard. Sources: S1, S2.
6. Configure the application's message-status webhook and inbound-message webhook when those callbacks are required. Success signal: The application contains the intended callback URLs. Sources: S1, S2.
7. Generate a JWT for the Vonage Application and prepare the `Bearer` Authorization header. Success signal: A non-expired JWT is available for the Authorization header. Sources: S1, S3.
8. Build the JSON request with `channel: sms`, `message_type: text`, message `text`, `from`, and `to`. Success signal: All required fields satisfy the API schema. Sources: S1, S3.
9. POST the JSON body to `https://{api-region}.nexmo.com/v1/messages` with the JWT and JSON content type. Success signal: The API accepts the request. Sources: S3.
10. Inspect the accepted response and capture its `message_uuid`. Success signal: The response is `Accepted` and includes `message_uuid`. Sources: S3.
