**Developer goal:** Follow Esri's general developer getting-started guidance to create credentials and display the first documented map.

**Meaningful success:** In the JavaScript candidate, the CodePen application displays the topographic basemap for the Santa Monica Mountains and allows pan and zoom.

**Evidence boundary:** explicitly-named. The general page labels Display a map as a first-application tutorial, and the JavaScript tutorial ends its required steps by running the app and observing the named topographic map before listing later tutorials.

## Documented path

1. Open Esri's mapping and location services Get started page. Success signal: The page offers ArcGIS Location Platform and ArcGIS Online accounts, API key and OAuth routes, and multiple Display a map clients. Sources: S1.
2. Open ArcGIS Location Platform signup and complete the email form with the documented identity, portal URL, security question, agreement, and privacy inputs. Success signal: The signup form is complete and Esri sends activation instructions. Sources: S2.
3. Follow the confirmation-email instructions within 24 hours. Success signal: The account can sign in to the Location Platform dashboard. Sources: S2.
4. Go to `https://location.arcgis.com`, sign in, and click My portal from the dashboard. Success signal: Portal content management is available. Sources: S3.
5. Click Content > My content > New item, then Developer credentials > API key credentials > Next. Success signal: The application-type screen is shown. Sources: S3.
6. Choose Public application and click Next, then select No item access and click Next. Success signal: The service-privilege screen opens. Sources: S3.
7. Select Location services > Basemaps, then click Next. Success signal: The expiration and referrer screen opens. Sources: S3, S4.
8. Set the expiration date to one month from today, enter allowed referrer domains, and click Next. Success signal: The item-details screen opens. Sources: S3.
9. Set title `My API key credentials`, create or select folder `Developer credentials`, add tags and a description, click Next, review the summary, and click Next again. Success signal: The Generate API key window appears. Sources: S3.
10. Select Generate the API key and go to item details page, confirm readiness to copy and save it, click Next, and copy the displayed key. Success signal: The Copy your key window displays the token and it is retained for the application. Sources: S3.
11. Create a new CodePen pen and add the tutorial's basic HTML and full-page CSS. Success signal: The HTML body is ready for the map component. Sources: S4.
12. Define global `esriConfig` before the SDK libraries and set its `apiKey` property to the copied access token. Success signal: `esriConfig.apiKey` contains the token value. Sources: S4.
13. Add `<script type="module" src="https://js.arcgis.com/5.1/"></script>` in the head. Success signal: The custom map elements are available to the page. Sources: S4.
14. Add an `arcgis-map` component with basemap `arcgis/topographic`, center `-118.805, 34.020`, and zoom `13`, then add an `arcgis-zoom` component in the top-left slot. Success signal: The page contains the complete tutorial markup. Sources: S4.
15. Run the CodePen application. Success signal: The Santa Monica Mountains basemap displays and can be panned and zoomed. Sources: S4.

## Documented friction gates

- Account, authentication, and client choices materially change the route and no universal combination is selected. Stage 2. Evidence state: documented requirement. Sources: S1.
- Email signup requires unique account and portal identifiers, a security answer, legal acceptance, and email activation. Stage 2. Evidence state: documented requirement. Sources: S2.
- The key must have the Basemap styles service privilege. Stage 2. Evidence state: documented requirement. Sources: S3, S4.
- The generated API key is visible only once and later regeneration invalidates the prior key. Stage 2. Evidence state: documented requirement. Sources: S3.
- Expiration and referrer restrictions are configured before generation, and changing them later invalidates the key. Stage 2. Evidence state: documented requirement. Sources: S3.
