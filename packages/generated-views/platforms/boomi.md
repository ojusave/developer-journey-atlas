**Developer goal:** Choose the relevant Boomi product surface and, for the Data Integration candidate, create the first source-to-target data flow and run it successfully.

**Meaningful success:** The developer creates reusable source and target connections, configures and saves a Source to Target Flow, runs it manually, and sees a successful run in the Activity panel.

**Evidence boundary:** demonstrated-terminal-state. A successful manual run is the first explicit, observable terminal in the selected Data Integration quick start. Scheduling and Kits follow it. A human still must confirm that Data Integration, rather than core Integration or another product, is the representative Boomi surface.

## Documented path

1. Open Boomi's product documentation and identify the product surface that owns the intended first outcome. Success signal: The workload can be mapped to a named Boomi product's getting-started documentation. Sources: S1, S2, S3.
2. For the recorded candidate, choose Data Integration and its Data Integration Flow quick-start guide. Success signal: The Set up your account step is the next documented action. Sources: S1.
3. Navigate to the Data Integration Console and select Start for Free. Success signal: Email and password, Google sign-in, and operational-region controls are available. Sources: S1.
4. Create the account with an email address and unique password, choose the operational region, and select Get Started. Success signal: The profile and workspace setup screen opens. Sources: S1.
5. Enter the developer's name, company name, and Data Integration workspace name, then select Next. Success signal: Data Integration sends an email-verification message. Sources: S1.
6. Open the verification email from Data Integration and select its verification link. Success signal: The account is verified and can return to onboarding. Sources: S1.
7. Select Let's Go to enter the Data Integration onboarding page. Success signal: The left navigation, including Connections and Data Flows, is available. Sources: S1.
8. Open Connections from the left menu and select + New Connection for the source. Success signal: The source-system selector is visible. Sources: S1.
9. Select the intended source system from the list. Success signal: The required credential fields for that source appear. Sources: S1.
10. Enter the source system's required credentials. Success signal: All source-required credential fields are accepted. Sources: S1.
11. Enter a Connection Name that identifies the source and select Save. Success signal: The source connection is present for selection in Data Flows. Sources: S1.
12. Return to Connections and select + New Connection for the target. Success signal: The target-system selector is visible. Sources: S1.
13. Select the intended cloud data warehouse or data-lake target. Success signal: The required fields for the selected target appear. Sources: S1.
14. Enter the target's required access credentials and configuration. Success signal: All target-required fields are accepted. Sources: S1.
15. Enter a Connection Name that identifies the target and select Save. Success signal: Both required reusable connections are available in the workspace. Sources: S1.
16. Open Data Flows from the left menu and select + New Data Flow. Success signal: Source to Target Flow, Logic Flow, and REST Action types are offered. Sources: S1.
17. Choose Source to Target Flow. Success signal: Controls for source, target, transformations, and scheduling are available. Sources: S1.
18. Select and configure the saved source connection for the Data Flow. Success signal: The Data Flow identifies what source data to extract. Sources: S1.
19. Select and configure the saved target connection for the Data Flow. Success signal: The Data Flow identifies where extracted data will be loaded. Sources: S1.
20. Configure any transformation options needed for the selected source-to-target flow and leave scheduling for after the first manual run unless required by the form. Success signal: All required flow fields are complete. Sources: S1.
21. Select Save. Success signal: The Data Flow can be opened and exposes Run. Sources: S1.
22. Open the saved Data Flow and select Run to execute it manually. Success signal: A run appears in the Activity panel with a status and logs. Sources: S1.
23. Monitor the run status and logs in the Activity panel until the run finishes. Success signal: The Activity panel reports that the data flow ran successfully. Sources: S1.

## Documented friction gates

- Boomi has peer product surfaces with materially different first journeys and no documented platform-wide default. Stage 1. Evidence state: documented requirement. Sources: S1, S2, S3.
- The Data Integration candidate is a representative selection, not an official Boomi-wide canonical route. Stage 2. Evidence state: documented requirement. Sources: S1, S2.
- The candidate requires both source-system credentials and supported target-warehouse access before the first flow can be built. Stage 2. Evidence state: documented requirement. Sources: S1.
- On-premise or firewall-protected sources require outbound network access to Data Integration's designated IP addresses. Stage 2. Evidence state: documented requirement. Sources: S1.
- Password signup requires an email-verification handoff; Google sign-in skips it. Stage 2. Evidence state: documented requirement. Sources: S1.
- The first manual run must be monitored until the Activity panel reports a terminal status. Stage 2. Evidence state: documented requirement. Sources: S1.
