**Developer goal:** Obtain credentials for a PayPal sandbox REST app and make the first OAuth 2.0 token request.

**Meaningful success:** The PayPal sandbox OAuth endpoint accepts the app's client ID and client secret and returns a bearer access token with its validity duration.

**Evidence boundary:** explicitly-named. The token is the first successful response from PayPal's authenticated REST API path and is a shared prerequisite across orders, payments, subscriptions, invoicing, and disputes. Selecting a later API would impose a product-specific outcome and request payload.

## Documented path

1. Open Get started with PayPal REST APIs. Success signal: Get client ID and client secret is the first numbered section. Sources: S1.
2. Select Log in to Dashboard. Success signal: The developer reaches a PayPal login or account-creation screen. Sources: S1.
3. Log in to PayPal or complete sign-up, then reach the Developer Dashboard. Success signal: Developer Dashboard navigation is visible. Sources: S1.
4. Select Apps & Credentials. Success signal: The Default Application or Create App action is visible. Sources: S1.
5. Use the Default Application in the REST API apps section. Success signal: The application's client ID and client secret controls are visible. Sources: S1.
6. Copy the app's client ID and client secret. Success signal: Both credential values are available for the OAuth request. Sources: S1.
7. Select the cURL example in Get access token. Success signal: The POST request to /v1/oauth2/token is visible. Sources: S1.
8. Replace CLIENT_ID and CLIENT_SECRET in the cURL command with the sandbox app credentials. Success signal: No placeholder credential names remain. Sources: S1, S2.
9. Send the OAuth token request to PayPal's sandbox. Success signal: PayPal returns a JSON token response rather than an authentication error. Sources: S1, S2.
10. Verify the returned access token and its metadata. Success signal: The response contains access_token, token_type, and expires_in. Sources: S1, S2, S3.
