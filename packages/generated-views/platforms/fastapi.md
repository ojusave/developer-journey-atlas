**Developer goal:** Create and run the simplest FastAPI application and observe its first HTTP response.

**Meaningful success:** A locally running FastAPI development server returns the documented Hello World JSON object for a GET request to `/`.

**Evidence boundary:** explicitly-named. This is the first end-to-end proof that installation, import, application construction, route registration, server startup, HTTP handling, and serialization all work. Interactive docs and deployment follow afterward.

## Documented path

1. Create a project directory, create a virtual environment with `python -m venv .venv`, and activate it for the current shell. Success signal: The shell uses the Python interpreter inside `.venv`; the optional official check is `which python` or `Get-Command python`. Sources: S1, S3.
2. Install the documented standard FastAPI package with `pip install "fastapi\[standard\]"` in the active environment. Success signal: The installation completes and the `fastapi` command is available. Sources: S1, S3, S4.
3. Create `main.py` containing the official simplest application: import `FastAPI`, instantiate `app`, register `GET /`, and return `{'message': 'Hello World'}` from `root`. Success signal: The file is present in the working directory with the `app` object and route exactly defined. Sources: S2.
4. From the project directory, run `fastapi dev` and wait for application startup. Success signal: The terminal reports `Application startup complete` and the local server URL. Sources: S1, S2.
5. Open `http://127.0.0.1:8000` in a browser and inspect the returned JSON. Success signal: The browser shows `{'message': 'Hello World'}` as documented under `Check it`. Sources: S2.
