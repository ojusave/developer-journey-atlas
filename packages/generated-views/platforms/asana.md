**Developer goal:** Create or access an Asana workspace, generate a personal access token, obtain the workspace GID, and complete the Quick start's task create, read, update, and delete sequence.

**Meaningful success:** The intended terminal is an authenticated sequence that creates `Sample task`, retrieves it, renames it to `Modified task`, deletes it, and reaches the guide's readiness statement.

**Evidence boundary:** explicitly-named. The Quick start places Next steps after the completed create, read, update, and delete sequence.

## Documented path

1. Open Asana's Quick start guide. Success signal: The setup section is available. Sources: S1, S2, S3.
2. Create an Asana account using the official login or account route and enter an accessible workspace. Success signal: Asana and the developer console can be opened. Sources: S2, S4, S5.
3. Open the developer console, select Create new token, name it, create it, and securely copy the one-time PAT. Success signal: The token is displayed. Sources: S3.
4. Open `https://app.asana.com/api/1.0/workspaces` while logged in and copy the target workspace GID. Success signal: The endpoint returns `200` with visible workspaces. Sources: S3.
5. POST `/tasks` with the PAT, workspace GID, name `Sample task`, and assignee `me`. Success signal: The API returns `201 Created`. Sources: S3.
6. GET `/tasks/{task_gid}` with the PAT. Success signal: The API returns `200 OK` and `Sample task`. Sources: S3.
7. PUT `/tasks/{task_gid}` to rename the task `Modified task`. Success signal: The API returns `200 OK` with `Modified task`. Sources: S3.
8. DELETE `/tasks/{task_gid}`. Success signal: The API returns `200 OK` with an empty data object. Sources: S3.
