**Developer goal:** Obtain authorized Adobe Experience Platform API credentials and prove them by listing the standard XDM classes available to the organization.

**Meaningful success:** The developer is added as both developer and user for an Experience Platform product profile, creates an Adobe Developer Console project with Experience Platform API and OAuth Server-to-Server credentials, generates an access token, imports Adobe's displayed global Schema Registry classes cURL request into Postman, executes it, and receives standard XDM classes.

**Evidence boundary:** explicitly-named. It is Adobe's explicit end of the authentication tutorial and proves enterprise entitlement, administrator assignments, Developer Console integration, API key, organization ID, OAuth token, and a real Platform API response. Product-specific resource work follows later.

## Documented path

1. Sign in to Adobe Developer Console with an Adobe ID or select Create a new account and complete signup. Success signal: The correct organization's Developer Console is accessible. Sources: S1.
2. Add the developer to an Experience Platform product profile as a developer and as a user. Success signal: The developer appears in the profile and can create Platform integrations. Sources: S1.
3. Create an empty project in Adobe Developer Console, using the organization selector to confirm the intended organization. Success signal: Project Overview shows Add API. Sources: S1.
4. Select Add API, choose Adobe Experience Platform, select Experience Platform API, and select Next. Success signal: Authentication-type selection is displayed. Sources: S1.
5. Choose OAuth Server-to-Server, enter a meaningful credential name, and select Next. Success signal: Product-profile selection appears. Sources: S1.
6. Select AEP-Default-All-Users and any additional required product profiles, then select Save configured API. Success signal: The OAuth Server-to-Server project page shows Client ID and Organization ID. Sources: S1.
7. Copy API_KEY (Client ID) and ORG_ID, select Generate access token, and copy ACCESS_TOKEN. Success signal: All three required header values are available. Sources: S1.
8. Copy Adobe's displayed cURL request for GET https://platform.adobe.io/data/foundation/schemaregistry/global/classes, import it into Postman, substitute ACCESS_TOKEN, API_KEY, and ORG_ID, and execute it. Success signal: Postman receives a results array rather than an authentication error. Sources: S1.
9. Verify the Postman response resembles Adobe's example and includes standard XDM class entries. Success signal: The expected standard XDM class objects appear in results. Sources: S1.
