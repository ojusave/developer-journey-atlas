## Documented path

1. Open OpenShift Getting Started and follow the free Developer Sandbox path. Success signal: The Sandbox page is available. Sources: S1, S2.
2. Select Start your sandbox for free and sign in with a Red Hat account. Success signal: The Start using your sandbox action is shown. Sources: S4.
3. Complete the Sandbox phone-number validation required by Red Hat. Success signal: The Sandbox access flow can continue. Sources: S7.
4. Select Start using your sandbox, then select DevSandbox on the OpenShift Dedicated login page. Success signal: The dashboard landing page appears. Sources: S4.
5. Select the Red Hat OpenShift tile and choose Launch. Success signal: A Welcome to the Developer Perspective message appears. Sources: S4.
6. Open the recommended Get started with your Developer Sandbox learning path. Success signal: The Overview and ordered resources are visible. Sources: S2, S3.
7. In OpenShift Topology, open Add to Project and select Import from Git. Success signal: The form asks for a Git repository URL. Sources: S5.
8. Enter one of the official sample repository URLs. Success signal: OpenShift presents deployment configuration for the imported source. Sources: S5.
9. Select Deployment as Resource type and select Create. Success signal: The application icon appears in Topology. Sources: S5.
10. Complete the source deployment and change the application icon ring from light blue to dark blue. Success signal: The ring is dark blue. Sources: S5.
11. Select the external-link icon and open the source-built application in a browser. Success signal: The application opens in the browser; the lesson says the application has been built from source code. Sources: S5.
12. Continue to the terminal resource, return to Topology, open Add to Project, and select Container image. Success signal: The form requests a fully qualified image name. Sources: S5, S6.
13. Enter quay.io/rhdevelopers/intro-front as the fully qualified container image. Success signal: The form accepts the image name. Sources: S6.
14. Select Deployment as Resource type and select Create. Success signal: The deployment is created and the terminal resource moves to optional play, break, and learn-more suggestions. Sources: S6.
