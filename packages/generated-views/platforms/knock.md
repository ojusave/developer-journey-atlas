## Documented path

1. Create a Knock account and log into the Knock dashboard. Success signal: Dashboard loads and user is authenticated Sources: S2.
2. Add the Knock SDK to the backend application or prepare to use the API. Success signal: SDK available in project or API client ready Sources: S2, S4.
3. Retrieve secret API keys from the Knock dashboard and set them as environment variables. Success signal: Backend can authenticate API calls to Knock Sources: S2, S4.
4. Create a workflow in the Knock dashboard. Success signal: Workflow appears in dashboard and is editable Sources: S2.
5. Commit the workflow to the current environment to activate it. Success signal: Workflow ready to be triggered Sources: S2.
6. Trigger the workflow via the Knock API to send a notification. Success signal: API call returns success and notification is delivered Sources: S2.
7. Publish workflows to production environment when ready. Success signal: Notifications sent to real users in production Sources: S2.

## Documented friction gates

- User must create and log into a Knock account to access the dashboard. Stage 1. Evidence state: documented requirement. Sources: S2.
- Developer must obtain and correctly configure secret API keys for authentication. Stage 3. Evidence state: documented requirement. Sources: S2, S4.
- Workflow must have at least one channel step configured to send notifications. Stage 4. Evidence state: documented requirement. Sources: S2.
- Trigger API call requires correct recipients and data payload to successfully send notifications. Stage 6. Evidence state: documented requirement. Sources: S2.
