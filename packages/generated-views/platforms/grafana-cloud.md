## Documented path

1. Create the free Grafana Cloud account and stack, then select demo data, connect externally managed data, or send real telemetry into Grafana Cloud. Success signal: No common first-data signal exists at this choice point. Sources: S1, S4, S5.
2. Choose telemetry signal, source, and integration or collector method. Success signal: The selected path can terminate with data visible in Grafana. Sources: S1, S2, S3.
3. Create or sign into a free Grafana Cloud account and open its Grafana Cloud stack. Success signal: The Grafana Cloud stack opens. Sources: S2, S4.
4. Prepare the Linux machine with terminal and administrator access, and configure its firewall to allow outbound HTTPS on port 443 to `*.grafana.net`. Success signal: The developer can administer the host and its firewall permits outbound HTTPS on port 443 to `*.grafana.net`. Sources: S2.
5. In the Grafana Cloud stack, open Connections, search for Linux, select the Linux integration tile, and open Configuration details. Success signal: Configuration details for the Linux integration are visible. Sources: S2.
6. Choose and enter the hostname label that will identify this machine in Grafana Cloud. Success signal: The configuration accepts the hostname value. Sources: S2.
7. Keep the reduced metric set or open Advanced integration and choose the full metric set. Success signal: The desired metric option is reflected in the generated setup. Sources: S2.
8. Copy and run the generated commands on the Linux machine to install, configure, and start Grafana Alloy. Success signal: The installation and service-start commands finish successfully. Sources: S2.
9. Open Dashboards and select the Nodes dashboard in the Linux Node folder. Success signal: The Nodes dashboard opens. Sources: S2.
10. Wait for Grafana Alloy to complete its first scrape and send the host metrics. Success signal: Dashboard panels begin populating. Sources: S2.
11. Verify that the Nodes dashboard displays recent metrics for the configured Linux host. Success signal: Recent metrics for the named host are visible in the dashboard. Sources: S2.

## Documented friction gates

- A Grafana Cloud account and stack must exist; the account starts with a 14-day unlimited trial and then transitions to the restricted free-forever plan. Stage 1. Evidence state: documented requirement. Sources: S1, S2, S4.
- Installing Demo Data Dashboards requires permission to create data sources and dashboards, such as the built-in Administrator role. Stage 1. Evidence state: documented requirement. Sources: S5.
- Externally connect versus send, signal type, data source, and collection route must be selected. Stage 1. Evidence state: documented requirement. Sources: S1, S3.
- Linux integration needs command-line and administrator access to the monitored host. Stage 2. Evidence state: documented requirement. Sources: S2.
- The monitored host firewall must allow outbound HTTPS to Grafana Cloud. Stage 2. Evidence state: documented requirement. Sources: S2.
- Hostname label and reduced versus full metric set must be selected and inserted into Alloy configuration. Stage 2. Evidence state: documented requirement. Sources: S2.
- The dashboard remains empty until the collector reaches its first scrape interval, then fills over time. Stage 2. Evidence state: documented requirement. Sources: S2.
