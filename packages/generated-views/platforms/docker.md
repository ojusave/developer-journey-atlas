**Developer goal:** Install Docker Desktop, start the official welcome container from the CLI, and see its frontend on localhost:8080.

**Meaningful success:** The developer installs and completes Docker Desktop setup, runs docker/welcome-to-docker detached with host port 8080 mapped to container port 80, and opens the rendered frontend in a browser.

**Evidence boundary:** explicitly-named. It proves installation, daemon startup, CLI access, registry image pull, container creation, port publication, running state, and browser reachability. Application development and image publishing are explicitly later modules.

## Documented path

1. Open the Mac install guide, confirm the Mac runs a supported macOS release with at least 4 GB RAM, determine Apple silicon or Intel architecture, and quit tools that might call Docker in the background. Success signal: The developer has selected the Apple silicon or Intel download matching the supported Mac. Sources: S4.
2. Download the matching Docker Desktop for Mac installer from the install page or release notes. Success signal: The correct Docker.dmg is downloaded. Sources: S4.
3. Double-click Docker.dmg, then drag the Docker icon to the Applications folder. Success signal: Docker.app appears in the Applications folder. Sources: S4.
4. Double-click Docker.app in Applications to start Docker Desktop. Success signal: The Docker menu displays the Docker Subscription Service Agreement. Sources: S4.
5. Review the Docker Subscription Service Agreement and select Accept. Success signal: Docker Desktop advances to the installation-settings window. Sources: S4, S5.
6. Choose Use recommended settings or Use advanced settings, then select Finish and enter the macOS password if the chosen configuration requires it. Success signal: The setup window finishes and Docker Desktop starts its engine. Sources: S4.
7. Wait for Docker Desktop to finish initialization and report ready. Success signal: Docker Desktop is ready and the CLI can reach the engine. Sources: S1, S4, S8.
8. Run docker run -d -p 8080:80 docker/welcome-to-docker. Success signal: The command exits successfully with a container ID and the container remains running. Sources: S1, S2.
9. Open http://localhost:8080 in a browser. Success signal: The Docker welcome landing page loads from the container. Sources: S1.
10. Open Docker Desktop, select Containers, and inspect the running container's information. Success signal: The welcome container is listed as running and its logs, files, shell, or inspect data can be opened. Sources: S1.

## Documented friction gates

- The representative route requires a supported Mac and the correct Apple silicon or Intel installer; Windows and Linux are materially different paths. Stage 1. Evidence state: documented requirement. Sources: S1, S4, S6, S7.
- Installing Docker.app in Applications requires local write permission, a mounted installer, and successful macOS security checks. Stage 3. Evidence state: documented requirement. Sources: S4.
- Docker Desktop cannot run until the Docker Subscription Service Agreement is accepted. Stage 5. Evidence state: documented requirement. Sources: S4, S5.
- A paid subscription is required for professional use in larger organizations, government entities, and commercial use beyond free-tier limits. Stage 5. Evidence state: documented requirement. Sources: S4, S5.
- The developer must choose recommended or advanced settings; recommended settings require a macOS password and advanced settings expose three configuration decisions. Stage 6. Evidence state: documented requirement. Sources: S4.
- The Docker engine must finish starting before CLI calls work; sign-in remains conditional unless an administrator enforces it. Stage 7. Evidence state: documented requirement. Sources: S1, S8.
- Registry access and an available host port 8080 are needed for the exact command. Stage 8. Evidence state: documented requirement. Sources: S1, S2.
