**Developer goal:** Obtain self-service X API access and make the first read-only API v2 request.

## Documented path

1. Open the X Developer Platform overview and follow Quickstart into Getting Access. Success signal: The three-step sign-up, app, and credential guide is visible. Sources: S1, S2.
2. Go to console.x.com and sign in with the X account. Success signal: The Developer Agreement is presented. Sources: S2.
3. Review and accept the Developer Agreement and Policy. Success signal: The profile step becomes available. Sources: S2.
4. Complete the developer profile with basic information about API use. Success signal: The Developer Console dashboard is available. Sources: S2.
5. From the Developer Console dashboard, select New App. Success signal: App detail fields are visible. Sources: S2.
6. Enter the app name, description, and use case, then create the app. Success signal: The Console generates the app's keys and tokens. Sources: S2.
7. Immediately save the generated credentials, especially the Bearer Token. Success signal: The Bearer Token is available for the public-data request. Sources: S2.
8. Purchase X API credits upfront in the Developer Console. Success signal: The credit balance is displayed and sufficient for the first user read. Sources: S4.
9. Make the saved Bearer Token available to cURL as BEARER_TOKEN. Success signal: The Authorization header can interpolate the token. Sources: S2, S3.
10. Send the user-lookup GET request with cURL and the Bearer Authorization header. Success signal: The request returns a data object instead of an error. Sources: S3, S4.
11. Parse and verify the JSON data field contains the X Developers user. Success signal: The expected user id, name, and username appear in data. Sources: S3.
