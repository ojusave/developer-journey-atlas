**Developer goal:** Create an ElevenLabs Free account and personal API key, then run the official Python Quickstart until the generated speech plays through the speakers.

**Meaningful success:** The developer's Python script sends the published text-to-speech request and audibly plays the returned speech.

**Evidence boundary:** demonstrated-terminal-state. The ElevenAPI Quickstart defines its outcome as a working script that sends text and plays returned audio, then lists streaming, voice browsing, and cloning as next steps.

## Documented path

1. Open the ElevenLabs Account guide and follow its Sign up link. Success signal: The documented email/password and OAuth registration choices are available. Sources: S1.
2. Choose traditional email registration and provide a personal email address and password to complete account registration for the non-commercial Free path. Success signal: The verification email is the documented next state before service use. Sources: S1, S7, S8.
3. Open the ElevenLabs email and select its verification link. Success signal: The Speech Synthesis page is the documented post-verification state. Sources: S1.
4. Assign the new account to the Free tier automatically. Success signal: No paid-plan selection is required before the documented API-key path. Sources: S5, S6.
5. Open Developers, select API Keys, create a named personal key, keep it restricted, and enable Text to Speech access. Success signal: The full key is shown for its one-time copy. Sources: S2, S3, S4.
6. Copy the full API key at creation and store it as a managed secret named `ELEVENLABS_API_KEY` in an `.env` file. Success signal: The `.env` file contains `ELEVENLABS_API_KEY=<your_api_key_here>`. Sources: S2, S3.
7. Install the Python SDK and dotenv loader with `pip install elevenlabs` and `pip install python-dotenv`. Success signal: The Quickstart imports are available for the example file. Sources: S2.
8. Create `example.py` with the published imports, load the `.env` file, and initialize `ElevenLabs` from `ELEVENLABS_API_KEY`. Success signal: The client is ready for the published text-to-speech conversion. Sources: S2.
9. Add the published `text_to_speech.convert` call with the sample sentence, George voice ID, `eleven_v3`, and `mp3_44100_128`, then pass the returned audio to `play`. Success signal: `example.py` contains the complete Quickstart request and playback call. Sources: S2.
10. Run `python example.py`. Success signal: The developer hears the generated audio through the speakers. Sources: S2, S6.

## Documented friction gates

- Email registration requires selecting the verification link before the service can be used; no link timing or recovery is published. Stage 2. Evidence state: documented requirement. Sources: S1.
- Users must be at least 18 and at least the age of legal majority where they live; free use is non-commercial, and a personal email is required for non-commercial use. Stage 2. Evidence state: documented requirement. Sources: S7, S8.
- The full personal API key is visible only at creation, and ElevenLabs recommends feature restrictions and secrecy. Stage 5. Evidence state: documented requirement. Sources: S3, S4.
- Speaker playback may require MPV and/or ffmpeg in addition to the Python packages. Stage 7. Evidence state: documented requirement. Sources: S2.
- The new account is on the Free tier, and each API request deducts usage from its included quota. Stage 10. Evidence state: documented requirement. Sources: S5, S6.
