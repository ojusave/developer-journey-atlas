## Documented path

1. Open the official Harness CI onboarding guide. Success signal: The guide exposes Create your first CI pipeline. Sources: S1.
2. Sign up for Harness as one opaque account step. Success signal: The Harness console opens for the new account. Sources: S3, S4.
3. Open the Continuous Integration module, select Pipelines, and choose Create a Pipeline. Success signal: Name and storage controls are visible. Sources: S1, S3.
4. Name the pipeline, choose Inline storage, and select Start. Success signal: Add Stage is available. Sources: S1, S3.
5. Add a Build stage, select the source provider and repository, and create or select a connector if required. Success signal: Infrastructure configuration becomes available. Sources: S1.
6. Choose Harness Cloud build infrastructure and continue with the documented operating-system and architecture defaults. Success signal: The Execution section opens. Sources: S1, S2.
7. In Execution, add a Run step and enter `echo "Harness CI is fast"`, then apply and save. Success signal: The Run step is visible in the saved stage. Sources: S1.
8. Select Run, choose Git Branch, enter the branch, and select Run Pipeline. Success signal: The execution view and build logs appear. Sources: S1.
9. Inspect the completed execution and Run-step logs. Success signal: The execution succeeds and the logs contain `Harness CI is fast`. Sources: S1, S2.
